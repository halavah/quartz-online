项目架构设计讨论

时间: 2024-01-15 14:30:00

标签: 架构, 设计, 微服务

摘要:

讨论了将单体应用重构为微服务架构的方案。主要包括以下几个方面：

  • 服务拆分策略：按业务领域划分服务边界
  • 数据一致性：采用事件溯源和CQRS模式
  • 服务通信：使用REST API和消息队列
  • 部署方案：容器化部署，使用Kubernetes编排

关键决策:

  1. 优先拆分用户服务和订单服务
  2. 引入API网关统一管理请求路由
  3. 采用分布式缓存提高性能

核心功能特性

为 AI 助手提供企业级的记忆管理能力

🧠

持久化记忆

分层记忆检索，模拟人类记忆模式，跨会话上下文保留，无缝衔接工作流程

🗜️

智能压缩

AI驱动的语义摘要生成，渐进式信息披露策略，记忆权重衰减和时间衰减机制

🔍

全文搜索

支持全文关键词搜索，语义搜索功能，理解查询意图，智能相关性排序

🌐

Web界面

在 http://localhost:37777 实时查看记忆流，可视化管理和浏览所有记忆内容

📊

统计分析

记忆统计和分析功能，使用趋势分析，记忆类型分布，性能监控

🤖

自动化管理

无需手动干预，后台透明运行，自动完成所有记忆管理工作，智能触发和更新

系统架构

模块化设计，协同工作完成记忆管理

1
会话开始 → 自动注入上下文
新会话开始时，系统自动检索并注入最近的观察和相关记忆作为上下文
↓
2
工具使用 → 自动捕获观察
实时捕获 Claude Code 的工具使用观察，记录重要操作和对话内容
↓
3
智能处理 → AI 压缩生成摘要
使用 AI 模型对捕获的信息进行智能压缩，生成语义摘要和关键词
↓
4
持久存储 → 分层存储记忆
将处理后的记忆内容分层存储到数据库，建立索引和关联关系
↓
5
会话结束 → 记忆保存完成
会话结束后，所有记忆内容完成保存，为下次会话提供上下文支持

安装指南

简单安装，快速开始使用持久化记忆功能

🔌
插件安装（推荐）
安装命令 
# 添加官方插件市场
claude plugin marketplace add https://github.com/anthropics/claude-code

# 安装 Claude-Mem 插件
claude plugin install claude-mem
🔧
手动安装
编译安装 
# 克隆仓库
git clone https://github.com/thedotmack/claude-mem.git
cd claude-mem

# 安装依赖
npm install

# 构建项目
npm run build
MCP 配置 (.claude/settings.json) 
{
  "mcpServers": {
    "claude-mem": {
      "command": "node",
      "args": ["/path/to/claude-mem/dist/index.js"],
      "env": {
        "CLAUDE_MEM_PORT": "37777",
        "CLAUDE_MEM_HOST": "localhost",
        "CLAUDE_MEM_DB_PATH": "./memory.db",
        "CLAUDE_MEM_MAX_TOKENS": "1000",
        "CLAUDE_MEM_COMPRESSION_MODEL": "claude-3-haiku-20240307"
      }
    }
  }
}

高级配置选项

配置项 描述 默认值
CLAUDE_MEM_PORT Web界面端口 37777
CLAUDE_MEM_HOST 服务器主机 localhost
CLAUDE_MEM_DB_PATH 数据库文件路径 ./memory.db
CLAUDE_MEM_MAX_TOKENS 最大Token数量 1000
CLAUDE_MEM_COMPRESSION_MODEL 压缩模型 claude-3-haiku-20240307
CLAUDE_MEM_RETENTION_DAYS 记忆保留天数 30
CLAUDE_MEM_AUTO_SUMMARIZE 自动摘要 true
CLAUDE_MEM_DEBUG 调试模式 false
