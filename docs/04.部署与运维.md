# 04. 部署与运维

## 🚀 快速部署到 Vercel（推荐）

### 方法 1: 通过 GitHub 集成

1. **推送代码到 GitHub**：
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/your-username/your-repo.git
   git push -u origin main
   ```

2. **在 Vercel 导入项目**：
   - 访问 [vercel.com](https://vercel.com)
   - 点击 "New Project"
   - 导入你的 GitHub 仓库
   - Vercel 会自动检测 Next.js 项目并配置
   - 点击 "Deploy"

### 方法 2: 使用 Vercel CLI

1. **安装 Vercel CLI**：
   ```bash
   npm i -g vercel
   ```

2. **登录并部署**：
   ```bash
   vercel login
   vercel --prod
   ```

## 🌐 部署到 Netlify

### 步骤 1: 构建项目

```bash
npm run build
```

### 步骤 2: 部署

1. 访问 [netlify.com](https://netlify.com)
2. 拖拽 `out` 文件夹到部署区域
3. 或连接 Git 仓库进行持续部署

### 配置设置

创建 `netlify.toml`：

```toml
[build]
  command = "npm run build"
  publish = "out"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 404
```

## 🔧 部署到静态文件服务器

### Nginx 配置

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    root /path/to/out;
    index index.html;

    # 处理 SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 缓存静态资源
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 压缩响应
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
}
```

### Apache 配置

创建 `.htaccess`：

```apache
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.html [L]

# 启用压缩
<IfModule mod_deflate.c>
    AddOutputFilterByType DEFLATE text/plain
    AddOutputFilterByType DEFLATE text/html
    AddOutputFilterByType DEFLATE text/xml
    AddOutputFilterByType DEFLATE text/css
    AddOutputFilterByType DEFLATE application/xml
    AddOutputFilterByType DEFLATE application/xhtml+xml
    AddOutputFilterByType DEFLATE application/rss+xml
    AddOutputFilterByType DEFLATE application/javascript
    AddOutputFilterByType DEFLATE application/x-javascript
</IfModule>
```

## ☁️ 部署到云存储

### AWS S3 + CloudFront

1. **上传到 S3**：
   ```bash
   aws s3 sync ./out s3://your-bucket-name --delete
   ```

2. **配置静态网站托管**：
   - 在 S3 控制台启用静态网站托管
   - 设置索引文档：`index.html`
   - 设置错误文档：`index.html`

3. **配置 CloudFront**：
   - 创建新的 CloudFront 分配
   - Origin 设置为 S3 bucket
   - 设置重定向规则

### 阿里云 OSS

```bash
# 使用 ossutil 上传
ossutil cp -r ./out oss://your-bucket-name/
```

### 腾讯云 COS

```bash
# 使用 COSCLI 上传
coscli sync ./out cos://your-bucket-name/
```

## 📦 部署到 CDN 平台

### Cloudflare Pages

1. 连接 GitHub 仓库
2. 配置构建设置：
   - 构建命令：`npm run build`
   - 输出目录：`out`
3. 部署！

### GitHub Pages

1. 修改 `next.config.ts`：
   ```typescript
   const nextConfig = {
     output: 'export',
     images: {
       unoptimized: true
     },
     basePath: '/repo-name', // 如果部署到子目录
     trailingSlash: true,
     assetPrefix: '/repo-name'
   };
   ```

2. 创建 `.github/workflows/deploy.yml`：
   ```yaml
   name: Deploy to GitHub Pages

   on:
     push:
       branches: [ main ]

   jobs:
     build-and-deploy:
       runs-on: ubuntu-latest
       steps:
       - uses: actions/checkout@v3
       - uses: actions/setup-node@v3
         with:
           node-version: '18'
       - run: npm ci
       - run: npm run build
       - uses: peaceiris/actions-gh-pages@v3
         with:
           github_token: ${{ secrets.GITHUB_TOKEN }}
           publish_dir: ./out
   ```

## 🐳 Docker 部署

### 创建 Dockerfile

```dockerfile
# 多阶段构建
FROM node:18-alpine AS builder

WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build

# 生产镜像
FROM nginx:alpine

# 复制构建产物
COPY --from=builder /app/out /usr/share/nginx/html

# 复制 nginx 配置
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

### 创建 nginx.conf

```nginx
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 缓存设置
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### 构建和运行

```bash
# 构建镜像
docker build -t halavah-tech-blog .

# 运行容器
docker run -d -p 80:80 halavah-tech-blog
```

## 🔧 常见问题与排查

### 问题 1: 构建失败

**症状**：`npm run build` 命令执行失败

**排查步骤**：
1. 检查 Node.js 版本（需要 18+）：
   ```bash
   node --version
   ```

2. 清除缓存并重新安装依赖：
   ```bash
   rm -rf .next node_modules package-lock.json
   npm install
   npm run build
   ```

3. 检查 TypeScript 错误：
   ```bash
   npx tsc --noEmit
   ```

### 问题 2: 路由 404 错误

**症状**：刷新页面或直接访问文章链接时出现 404

**解决方案**：
- **Vercel/Netlify**: 自动处理，无需配置
- **Nginx**: 添加重定向规则（见上文 Nginx 配置）
- **GitHub Pages**: 添加 404.html 重定向：
  ```html
  <!DOCTYPE html>
  <html>
  <head>
    <meta http-equiv="refresh" content="0;URL='/'">
  </head>
  </html>
  ```

### 问题 3: 图片不显示

**症状**：部署后图片无法加载

**排查步骤**：
1. 确认图片在 `public/` 目录下
2. 检查 `next.config.ts` 配置：
   ```typescript
   images: {
     unoptimized: true
   }
   ```
3. 检查图片路径（相对路径 vs 绝对路径）

### 问题 4: 样式丢失

**症状**：部署后页面样式不正常

**排查步骤**：
1. 检查 Tailwind CSS 配置：
   ```typescript
   // tailwind.config.ts
   content: [
     "./app/**/*.{js,ts,jsx,tsx,mdx}",
   ]
   ```

2. 确认 `globals.css` 包含 Tailwind 指令：
   ```css
   @tailwind base;
   @tailwind components;
   @tailwind utilities;
   ```

3. 清除构建缓存重新构建：
   ```bash
   rm -rf .next
   npm run build
   ```

### 问题 5: 搜索功能不工作

**症状**：点击搜索按钮没有反应

**排查步骤**：
1. 检查浏览器控制台是否有 JavaScript 错误
2. 确认 `articles` 数据正确传递给 `Navbar` 组件
3. 检查是否有 CSS 样式冲突导致弹窗不可见

### 问题 6: 主题切换不生效

**症状**：点击主题切换按钮，页面样式没有变化

**排查步骤**：
1. 检查浏览器 localStorage 是否可用
2. 确认 `globals.css` 中定义了两套主题 CSS 变量
3. 检查 `data-theme` 属性是否正确设置到 `<html>` 元素

### 问题 7: 分页不显示

**症状**：文章很多但没有分页器

**排查步骤**：
1. 检查 `totalPages` 是否大于 1
2. 确认 `Pagination` 组件正确引入
3. 检查条件渲染逻辑：
   ```tsx
   {totalPages > 1 && currentArticles.length > 0 && (
     <Pagination ... />
   )}
   ```

### 问题 8: 部署后性能差

**症状**：Lighthouse 评分低，加载慢

**优化措施**：
1. **启用 Gzip 压缩**（服务器配置）
2. **优化图片**：
   - 使用 WebP 格式
   - 压缩图片文件大小
   - 添加懒加载
3. **使用 CDN**：
   - 配置 CloudFront/Cloudflare
   - 缓存静态资源
4. **减少 JavaScript 大小**：
   - 移除未使用的依赖
   - 代码分割

## 🛠️ 运维工具和脚本

### 自动化部署脚本

创建 `scripts/deploy.sh`：

```bash
#!/bin/bash

echo "🚀 开始部署..."

# 1. 清理旧构建
echo "📦 清理旧构建..."
rm -rf .next out

# 2. 安装依赖
echo "📥 安装依赖..."
npm install

# 3. 构建项目
echo "🔨 构建项目..."
npm run build

# 4. 上传到服务器（示例：使用 rsync）
echo "📤 上传到服务器..."
rsync -avz --delete out/ user@server:/var/www/html/

echo "✅ 部署完成！"
```

使用方法：

```bash
chmod +x scripts/deploy.sh
./scripts/deploy.sh
```

### 备份脚本

创建 `scripts/backup.sh`：

```bash
#!/bin/bash

# 备份目录
BACKUP_DIR="backups/$(date +%Y%m%d_%H%M%S)"

mkdir -p $BACKUP_DIR

# 备份文章和配置
cp -r public/ $BACKUP_DIR/
cp -r data/ $BACKUP_DIR/
cp -r docs/ $BACKUP_DIR/

echo "✅ 备份完成：$BACKUP_DIR"
```

### 生成 Sitemap

创建 `scripts/generate-sitemap.js`：

```javascript
const fs = require('fs');
const articles = require('../data/articles.json');

const DOMAIN = 'https://yourdomain.com';

const sitemap = `<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>${DOMAIN}</loc>
    <changefreq>daily</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>${DOMAIN}/articles</loc>
    <changefreq>daily</changefreq>
    <priority>0.9</priority>
  </url>
  <url>
    <loc>${DOMAIN}/categories</loc>
    <changefreq>weekly</changefreq>
    <priority>0.8</priority>
  </url>
  <url>
    <loc>${DOMAIN}/about</loc>
    <changefreq>monthly</changefreq>
    <priority>0.7</priority>
  </url>
${articles.articles.map(article => `  <url>
    <loc>${DOMAIN}/article/${article.htmlFile.replace('.html', '')}</loc>
    <changefreq>monthly</changefreq>
    <priority>0.8</priority>
  </url>`).join('\n')}
</urlset>`;

fs.writeFileSync('public/sitemap.xml', sitemap);
console.log('✅ Sitemap 生成成功：public/sitemap.xml');
```

使用方法：

```bash
node scripts/generate-sitemap.js
```
