## 概述

### 优化目标
- **TTFB**: 从 800-1200ms 降低到 400-600ms
- **首屏加载**: 从 3-5s 降低到 1.5-2.5s
- **HTML 加载**: 从 35-40KB/500ms 降低到 35-40KB/200ms

### 优化策略
1. 启用 Auto Minify (自动压缩)
2. 配置 Page Rules (页面规则)
3. 启用 HTTP/3 和 0-RTT
4. 优化缓存策略
5. 可选: Argo Smart Routing (智能路由)

## 基础配置

### 1. DNS 设置

1. 登录 Cloudflare Dashboard
2. 进入 **DNS** 页面
3. 确保你的域名记录已添加:
   ```
   Type: A/CNAME
   Name: @ (或 www)
   Content: 你的 Vercel 部署地址
   Proxy status: Proxied (橙色云朵)
   TTL: Auto
   ```

### 2. SSL/TLS 模式

1. 进入 **SSL/TLS** 页面
2. 选择 **Full (strict)** 模式
   - 这确保了端到端的 SSL 加密
   - Vercel 支持严格的 SSL 证书验证

## 性能优化

### 3. Auto Minify (自动压缩)

**路径**: `Speed` → `Optimization` → `Auto Minify`

**推荐设置**:
- ✅ JavaScript
- ✅ CSS
- ✅ HTML

**效果**:
- JavaScript: 减少 10-20% 体积
- CSS: 减少 15-25% 体积
- HTML: 减少 5-10% 体积 (本项目 HTML 35-40KB → 32-36KB)

### 4. Brotli 压缩

**路径**: `Speed` → `Optimization` → `Brotli`

**设置**: ✅ 启用

**效果**:
- 比 Gzip 额外减少 15-20% 体积
- HTML 文件: 35-40KB → 8-12KB (压缩后)
- 浏览器自动支持，无需额外配置

### 5. Early Hints

**路径**: `Speed` → `Optimization` → `Early Hints`

**设置**: ✅ 启用

**效果**:
- 浏览器提前知道需要加载的资源
- 减少 100-300ms 的资源发现时间
- 对 HTML 预加载特别有效

### 6. HTTP/3 (QUIC)

**路径**: `Network` → `HTTP/3 (with QUIC)`

**设置**: ✅ 启用

**效果**:
- 更快的连接建立 (0-RTT)
- 更好的弱网络环境性能
- 减少握手时间 50-100ms

### 7. 0-RTT Connection Resumption

**路径**: `Network` → `0-RTT Connection Resumption`

**设置**: ✅ 启用

**效果**:
- 回访用户立即建立连接 (0 RTT)
- 首次请求延迟降低 100-200ms
- 对中国用户特别有效 (延迟高)

### 8. WebSockets

**路径**: `Network` → `WebSockets`

**设置**: ✅ 启用

**原因**: 虽然当前项目不使用 WebSocket，但保持启用以备将来需求

## 缓存策略

### 9. Page Rules (关键配置)

**路径**: `Rules` → `Page Rules` → `Create Page Rule`

#### 规则 1: HTML 文件缓存
```
URL pattern: *yourdomain.com/*.html

Settings:
- Cache Level: Cache Everything
- Edge Cache TTL: 1 month
- Browser Cache TTL: 1 day
```

**说明**:
- HTML 文件是静态的，可以长期缓存
- 边缘节点缓存 1 个月 (2592000 秒)
- 浏览器缓存 1 天 (86400 秒)

#### 规则 2: 静态资源缓存
```
URL pattern: *yourdomain.com/_next/static/*

Settings:
- Cache Level: Cache Everything
- Edge Cache TTL: 1 year
- Browser Cache TTL: 1 year
```

**说明**:
- Next.js 静态资源包含内容哈希，可以永久缓存
- 文件变化时 URL 会改变，自动失效

#### 规则 3: 文章页面缓存
```
URL pattern: *yourdomain.com/chapter*/*

Settings:
- Cache Level: Cache Everything
- Edge Cache TTL: 1 week
- Browser Cache TTL: 1 day
```

**说明**:
- 文章页面是动态路由但内容静态
- 边缘节点缓存 1 周 (604800 秒)
- 浏览器缓存 1 天以便快速更新

### 10. Caching Configuration

**路径**: `Caching` → `Configuration`

**推荐设置**:
- **Caching Level**: Standard
- **Browser Cache TTL**: Respect Existing Headers
- **Always Online**: ✅ 启用 (CDN 缓存作为备份)

## 安全设置

### 11. Security Level

**路径**: `Security` → `Settings` → `Security Level`

**设置**: Medium

**原因**: 平衡安全性和用户体验，Medium 级别适合博客网站

### 12. Bot Fight Mode

**路径**: `Security` → `Bots` → `Bot Fight Mode`

**设置**: ✅ 启用

**效果**: 自动过滤恶意机器人，减少服务器负载

### 13. Hotlink Protection (可选)

**路径**: `Scrape Shield` → `Hotlink Protection`

**设置**: 根据需求启用

**说明**: 防止其他网站直接链接你的 HTML 文件

## 高级优化 (可选)

### 14. Argo Smart Routing

**路径**: `Traffic` → `Argo Smart Routing`

**费用**: $5/月 + $0.10/GB

**效果**:
- 智能选择最优路由
- 平均减少 30% 延迟
- 对中国用户特别有效 (可减少 200-500ms)

**推荐**: 如果预算允许，强烈推荐开启

**设置步骤**:
1. 点击 **Enable Argo**
2. 确认付费计划
3. 等待 5-10 分钟生效

### 15. Tiered Cache Topology

**路径**: `Caching` → `Tiered Cache`

**设置**: ✅ 启用

**效果**:
- 多层缓存架构
- 降低回源请求
- 提高缓存命中率

**说明**: 只在 Pro 或以上套餐可用

## 监控与测试

### 性能指标

在 Cloudflare Dashboard 的 **Analytics** 页面查看:
- **Bandwidth**: 流量使用情况
- **Requests**: 请求数统计
- **Cache Hit Rate**: 缓存命中率 (目标: >90%)
- **Status Codes**: HTTP 状态码分布

### 中国访问测试

1. **本地测试**
   ```bash
   # 测试 TTFB
   curl -o /dev/null -s -w "TTFB: %{time_starttransfer}s\nTotal: %{time_total}s\n" \
     https://yourdomain.com/chapter01/0101.windows-installation.html
   ```

2. **在线多地点测试**
   - https://www.17ce.com/ (国内多节点测试)
   - 选择多个中国城市节点
   - 测试目标页面加载速度

3. **Chrome DevTools 性能分析**
   - 打开 Chrome DevTools (F12)
   - 切换到 **Network** 标签
   - 勾选 **Disable cache**
   - 刷新页面
   - 查看 **TTFB** 和 **Load** 时间

## 故障排除

### 问题 1: 缓存未生效

**症状**: `cf-cache-status: MISS` 始终显示

**解决方案**:
1. 检查 Page Rules 是否正确配置
2. 确认 URL pattern 匹配
3. 等待 5-10 分钟让规则生效
4. 使用 **Purge Cache** 清除所有缓存后重试

### 问题 2: TTFB 仍然很高

**症状**: TTFB > 800ms

**解决方案**:
1. 确认 HTTP/3 和 0-RTT 已启用
2. 检查 Vercel 部署地区 (应该选择 HKG1 - Hong Kong)
3. 考虑启用 Argo Smart Routing
4. 检查是否有大量未缓存的动态请求

### 问题 3: HTML 未压���

**症状**: HTML 文件大小 35-40KB (未压缩)

**解决方案**:
1. ��认 Auto Minify - HTML 已启用
2. 确认 Brotli 已启用
3. 检查响应头: `content-encoding: br` 或 `gzip`
4. 清除缓存后重新测试

### 问题 4: 中国访问仍然很慢

**症状**: 总加载时间 > 3s

**解决方案**:
1. 确认所有 Page Rules 已配置
2. 检查是否有被 GFW 阻断的外部资源 (如 Google Fonts - 已移除)
3. 使用 Argo Smart Routing
4. 考虑使用国内 CDN 作为补充 (如阿里云、腾讯云)
