条件类型 - 类型层面的 if/else

中级
条件类型让我们能够根据类型关系来选择类型，这是类型编程的核心机制
基础条件类型
typescript
// 基础语法：T extends U ? X : Y
type IsString<T> = T extends string ? true : false

type Test1 = IsString<string>  // true
type Test2 = IsString<number>  // false

// 提取数组元素类型
type ArrayElement<T> = T extends (infer U)[] ? U : never

type Numbers = ArrayElement<number[]>  // number
type Strings = ArrayElement<string[]>  // string

// 多条件判断
type TypeName<T> =
  T extends string ? "string" :
  T extends number ? "number" :
  T extends boolean ? "boolean" :
  T extends undefined ? "undefined" :
  T extends Function ? "function" :
  "object"

映射类型 - 动态创建类型

中级
映射类型允许我们遍历类型的属性并创建新类型，是构建灵活类型系统的关键
高级映射类型
typescript
// 深度 Partial
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P]
}

// 深度 Readonly
type DeepReadonly<T> = {
  readonly [P in keyof T]: T[P] extends object ? DeepReadonly<T[P]> : T[P]
}

// 条件映射
type ConditionalPartial<T, K extends keyof T> =
  Partial<Pick<T, K>> & Omit<T, K>

// 使用示例
interface User {
  id: number
  name: string
  contact: {
    email: string
    phone: string
  }
}

type PartialUser = DeepPartial<User>
// 所有属性都变为可选，包括嵌套对象

模板字面量类型 - 字符串魔法

高级
TypeScript 4.1+ 引入的模板字面量类型，让字符串操作在类型层面成为可能
高级模板操作
typescript
// 基础模板类型
type Greeting = `Hello, ${string}!`
// "Hello, World!", "Hello, TypeScript!" 等

// 动态属性名
type EventHandlers<T> = {
  [K in keyof T as K extends string
    ? `on${Capitalize<K>}`
    : never]: T[K] extends (...args: any[]) => any
      ? T[K]
      : (event: any) => void
}

// 使用示例
interface Events {
  click: (e: MouseEvent) => void
  scroll: (e: Event) => void
}

type Handlers = EventHandlers<Events>
// { onClick: (e: MouseEvent) => void; onScroll: (e: Event) => void }

// 路径类型
type Path<T> = T extends object
  ? {
      [K in keyof T]: K extends string
        ? T[K] extends object
          ? `${K}.${Path<T[K]>}`
          : K
        : never
    }[keyof T]
  : never

interface Nested {
  user: {
    name: string
    address: {
      street: string
    }
  }
}

type NestedPaths = Path<Nested>
// "user" | "user.name" | "user.address" | "user.address.street"

递归类型 - 无限可能

高级
递归类型让我们能够定义自引用的类型，处理任意深度的数据结构
递归模式示例
typescript
// JSON 类型定义
type JSONValue =
  | string
  | number
  | boolean
  | null
  | JSONObject
  | JSONArray

interface JSONObject {
  [key: string]: JSONValue
}

interface JSONArray extends Array<JSONValue> {}

// 树结构类型
interface TreeNode<T> {
  value: T
  left?: TreeNode<T>
  right?: TreeNode<T>
}

// 嵌套数组类型
type NestedArray<T> = T | NestedArray<T>[]

// 使用示例
const data: NestedArray<number> = [1, [2, [3, 4]], 5]

// 递归工具类型
type Flatten<T> = T extends readonly (infer U)[]
  ? U extends readonly unknown[]
    ? Flatten<U>
    : U
  : T;

type FlattenNested = Flatten<NestedArray<number>>;  // number

类型安全的 API 客户端

高级
构建一个完全类型安全的 API 客户端，自动生成请求和响应类型
类型安全 API 客户端实现
typescript
// API 路径类型
type APIRoutes = {
  '/users': {
    get: { response: User[] }
    post: { body: CreateUserDto; response: User }
  }
  '/users/:id': {
    get: { params: { id: string }; response: User }
    put: { params: { id: string }; body: UpdateUserDto; response: User }
    delete: { params: { id: string }; response: void }
  }
}

// 动态生成客户端方法
type APIClient<T> = {
  [K in keyof T]: T[K] extends { get: any }
    ? (params?: T[K]['get']['params']) => Promise<T[K]['get']['response']>
    : never
} & {
  [K in keyof T as T[K] extends { post: any }
    ? `post${Capitalize<K & string>}`
    : never]: T[K] extends { post: any }
      ? (body: T[K]['post']['body']) => Promise<T[K]['post']['response']>
      : never
}

// 使用示例
const client: APIClient<APIRoutes> = {
  '/users': async () => {
    const response = await fetch('/api/users')
    return response.json()
  },
  '/users/:id': async (params) => {
    const response = await fetch(`/api/users/${params?.id}`)
    return response.json()
  },
  postUsers: async (body) => {
    const response = await fetch('/api/users', {
      method: 'POST',
      body: JSON.stringify(body)
    })
    return response.json()
  }
}

// 完全类型安全的使用
const users = await client['/users']()  // User[]
const user = await client['/users/:id']({ id: '1' })  // User
const newUser = await client.postUsers({ name: 'John' })  // User

最佳实践总结

使用 TypeScript 高级类型的建议和注意事项

✅ 推荐做法：
  • 保持类型简单明了，避免过度设计
  • 充分利用 TypeScript 的类型推断
  • 使用 JSDoc 注释解释复杂类型
  • 定期重构和优化类型定义
  • 使用类型体操解决实际问题，而非炫技
⚠️ 避免做法：
  • 创建过于复杂的类型，影响编译性能
  • 过度使用 any，失去类型安全
  • 忽略类型错误，使用类型断言绕过检查
  • 不合理的类型嵌套，导致可读性差
性能优化示例
typescript
// ❌ 避免过度的条件类型嵌套
type BadType<T> = T extends string
  ? T extends `${infer A}-${infer B}`
    ? B extends number
      ? `${A}${Capitalize<B>}`
      : never
    : never
  : never

// ✅ 使用辅助类型简化
type SplitDash<T> = T extends `${infer A}-${infer B}` ? [A, B] : never
type IsNumber<T> = T extends number ? true : false
type OptimizeType<T> = T extends string
  ? SplitDash<T> extends [string, infer B]
    ? IsNumber<B> extends true
      ? `${SplitDash<T>[0]}${Capitalize<B>}`
      : never
    : never
  : never
