# Vercel 配置说明

## 📋 配置概览

本文档说明 `vercel.json` 中的所有配置项及其作用。

---

## ⚙️ 基础配置

### 构建设置
```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "out",
  "installCommand": "npm install",
  "framework": "nextjs"
}
```

- **buildCommand**: 构建命令，执行 Next.js 静态导出
- **outputDirectory**: 输出目录，Next.js 静态导出默认为 `out/`
- **installCommand**: 安装依赖命令
- **framework**: 指定框架类型，帮助 Vercel 优化构建

### URL 设置
```json
{
  "cleanUrls": true,
  "trailingSlash": true
}
```

- **cleanUrls**: 移除 URL 中的 `.html` 扩展名
  - `/about.html` → `/about`
- **trailingSlash**: 所有 URL 以斜杠结尾
  - `/about` → `/about/`

## 🔒 安全头部（Security Headers）

### 全局安全设置
应用于所有路由 `/(.*)`：

| Header | 值 | 作用 |
|--------|-----|------|
| `X-Content-Type-Options` | `nosniff` | 防止 MIME 类型嗅探攻击 |
| `X-Frame-Options` | `DENY` | 防止点击劫持（Clickjacking） |
| `X-XSS-Protection` | `1; mode=block` | 启用浏览器 XSS 保护 |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | 控制 Referrer 信息泄露 |
| `Permissions-Policy` | `camera=(), microphone=()...` | 禁用不必要的浏览器 API |

## ⚡ 缓存策略（Cache Control）

### 静态资源缓存
```json
{
  "source": "/(.*)\\.(?:jpg|jpeg|gif|png|svg|ico|webp|avif)",
  "headers": [
    {
      "key": "Cache-Control",
      "value": "public, max-age=31536000, immutable"
    }
  ]
}
```

**���存时间：1年（31536000 秒）**

#### 应用于：
- **图片文件**: `.jpg`, `.jpeg`, `.gif`, `.png`, `.svg`, `.ico`, `.webp`, `.avif`
- **脚本和样式**: `.js`, `.css`
- **Public 目录**: `/public/` 下的所有文件

#### 为什么这样设置？
- `public`: 允许 CDN 和浏览器缓存
- `max-age=31536000`: 缓存 1 年
- `immutable`: 告诉浏览器该资源永不改变

## 🔀 重定向规则（Redirects）

### 当前配置
```json
{
  "source": "/home",
  "destination": "/",
  "permanent": true
}
```

#### 用途
- 将 `/home` 永久重定向到首页 `/`
- `permanent: true` 表示 301 永久重定向（对 SEO 友好）

#### 如何添加更多重定向
```json
{
  "redirects": [
    {
      "source": "/old-path",
      "destination": "/new-path",
      "permanent": true
    },
    {
      "source": "/blog/:slug",
      "destination": "/articles/:slug",
      "permanent": false
    }
  ]
}
```

## 🔄 重写规则（Rewrites）

### 当前配置
```json
{
  "source": "/sitemap.xml",
  "destination": "/api/sitemap"
}
```

#### 用途
- 将 `/sitemap.xml` 请求转发到 API 路由
- 用户看到的 URL 不变，但实际请求被重写

#### 重写 vs 重定向
- **重写（Rewrite）**: URL 不变，后���处理不同路由
- **重定向（Redirect）**: URL 改变，浏览器跳转

## 🌏 区域设置（Regions）

```json
{
  "regions": ["hkg1", "sin1"]
}
```

### 选择的区域
- **hkg1**: 香港数据中心
- **sin1**: 新加坡数据中心

### 为什么选择这些区域？
- 针对亚洲用户优化
- 降低延迟
- 提高访问速度

### 其他可用区域
| 代码 | 位置 | 适用地区 |
|------|------|----------|
| `hkg1` | 香港 | 东亚、东南亚 |
| `sin1` | 新加坡 | 东南亚 |
| `icn1` | 首尔 | 韩国 |
| `nrt1` | 东京 | 日本 |
| `sfo1` | 旧金山 | 北美西海岸 |
| `iad1` | 华盛顿 | 北美东海岸 |
| `cdg1` | 巴黎 | 欧洲 |

## 🔧 Git 部署配置

```json
{
  "git": {
    "deploymentEnabled": {
      "main": true,
      "master": true
    }
  }
}
```

### 自动部署分支
- **main**: 主分支自动部署
- **master**: 主分支自动部署（兼容旧项目）

### 如何禁用自动部署
```json
{
  "git": {
    "deploymentEnabled": {
      "main": false
    }
  }
}
```

## 📦 还需要配置的内容

### 1. 环境变量
在 Vercel 控制面板中设置：
- `NEXT_PUBLIC_SITE_URL`: 网站 URL
- `NEXT_PUBLIC_GA_ID`: Google Analytics ID（如果使用）
- 其他 API 密钥

### 2. 自定义域名
在 Vercel 控制面板中添加：
1. 进入项目设置
2. 点击 "Domains"
3. 添加自定义域名
4. 配置 DNS 记录

### 3. 性能监控
启用 Vercel Analytics：
```bash
npm install @vercel/analytics
```

在 `app/layout.tsx` 中添加：
```typescript
import { Analytics } from '@vercel/analytics/react';

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        {children}
        <Analytics />
      </body>
    </html>
  );
}
```
