03.常见操作
    a.使用Terminal
        a.引用文件
            a.方法1：直接输入文件路径
                @/path/to/your/file.js
                或者
                @src/components/Button.tsx
            b.方法2：使用相对路径
                @./script/01-check-database.sh
                @worker/src/index.ts
            c.方法3：使用项目根目录
                @wrangler.toml
                @README.md
                @src/utils/helper.js
            d.方法4：搜索特定内容
                @function:deleteExpiredEmails
                @class:EmailService
        b.使用技巧
            a.多个文件：可以用空格分隔
                @src/App.js @src/index.css @package.json
            b.目录：使用通配符
                @src/components/*
                @script/*.sh
            c.特定行：加上行号
                @worker/src/index.ts:100-120
            d.类型过滤
                @*.js @*.ts @*.tsx
        c.智能提示
            Claude Code 会自动补全文件路径
            支持 Tab 键补全
            会显示相关文件的预览
    b.安装plugin
        a.添加Claude Code插件市场
            claude plugin marketplace add https://github.com/anthropics/claude-code
            -------------------------------------------------------------------------------------------------
            ⎿  Successfully added marketplace: claude-code-plugins
        b.验证市场安装
            # 查看已配置的市场
            claude plugin marketplace list
            -------------------------------------------------------------------------------------------------
            预期输出
            Configured marketplaces:
              ❯ claude-code-plugins
                Source: Git (https://github.com/anthropics/claude-code.git)
        c.安装 Frontend Design 插件
            # 安装 frontend-design 插件
            claude plugin install frontend-design
            -------------------------------------------------------------------------------------------------
            ✔ Successfully installed plugin: frontend-design@claude-code-plugins
    c.使用skills
        a.第1步
            在你的项目目录下创建一个 “.claude/skills”目录，如果你不想提交到 git 就把 .claude 加到 .gitignore
            注：也可以是任意其他目录，放在“.claude/skills”目录下有个好处就是 claude code 默认能使用，不需要额外配置。
        b.第2步
            把你要用到 skill 复制到“.claude/skills”目录下（可以去 http://github.com/anthropics/skills 这里找现成的）
        c.第3步
            如果你需要用到哪个 skill，只需要手动 @ 一下相应的 skill 文件即可，比如：
            > 请使用 @.claude/skills/artifacts-builder/SKILL.md ，创建一个 whiteboard 项目
        d.第4步
            也就是说只要你让 agent 去读取相应的 SKILL md 文件，就可以让 Agent 学会使用 SKILL。
            这个方法不仅仅适用于 codex cli，也同样适用于 TRAE、Cursor、GitHub Copilot 这类 coding agent。
            只能说 SKILL 的设计是想当超前的，而且跟 MCP 一样，并非 Claude Code 专属。

            claude -c -p "query"                    通过 SDK 继续                                    claude -c -p "Check for type errors"

            claude update                           更新至最新版本                                   claude update
            claude mcp                              配置模型上下文协议 (MCP) 服务器                  参见Claude Code MCP 文档。
        b.CLI标志
            | 旗帜                             | 描述                                                                   | 例子                                                        |
            |----------------------------------|------------------------------------------------------------------------|-------------------------------------------------------------|
            | --add-dir                        | 添加额外的工作目录供 Claude 访问（验证每个路径是否存在于目录中）         | claude --add-dir ../apps ../lib                             |

            | --output-format                  | 指定打印模式的输出格式（选项：text，，json）stream-json                  | claude -p "query" --output-format json                      |

    b.自定义斜线命令
        a.开始
            a.说明
                自定义斜线命令允许您将常用的提示定义为 Claude Code 可执行的 Markdown 文件。
                命令按范围（项目特定或个人）组织，并支持通过目录结构进行命名空间管理。
            b.句法
                /<prefix>:<command-name> [arguments]
            c.参数
                范围           描述
                <prefix>       命令范围（project针对项目特定，user针对个人）
                <command-name> 从 Markdown 文件名派生的名称（不带.md扩展名）
                [arguments]    传递给命令的可选参数
        b.命令类型
            a.项目命令：命令存储在您的存储库中并与您的团队共享。
                位置：.claude/commands/
                前缀：/project:
                ---------------------------------------------------------------------------------------------
                在下面的示例中，我们创建/project:optimize命令：
                # Create a project command
                mkdir -p .claude/commands
                echo "Analyze this code for performance issues and suggest optimizations:" > .claude/commands/optimize.md
            b.个人命令：所有项目均可使用的命令
                位置：~/.claude/commands/
                前缀：/user:
                ---------------------------------------------------------------------------------------------
                在下面的示例中，我们创建 /user:security-review命令：
                # Create a personal command
                mkdir -p ~/.claude/commands
                echo "Review this code for security vulnerabilities:" > ~/.claude/commands/security-rev
        c.特征
            a.命名空间
                组织子目录中的命令以创建命名空间命令。
                结构：<prefix>:<namespace>:<command>
                例如，文件 at.claude/commands/frontend/component.md创建命令/project:frontend:component
            b.参数
                使用占位符将动态值传递给命令$ARGUMENTS。
                例如：
                # Command definition
                echo "Fix issue #$ARGUMENTS following our coding standards" > .claude/commands/fix-issue.md
                # Usage
                > /project:fix-issue 123
        d.文件格式
            a.命令文件必须
                使用 Markdown 格式（.md扩展名）
                包含提示或说明作为文件内容
                放置在适当的命令目录中
    c.MCP 斜线命令
        a.开始
            a.说明
                MCP 服务器可以将提示符显示为斜杠命令，这些命令可在 Claude 代码中使用。这些命令是从已连接的 MCP 服务器动态发现的。
            b.命令格式
                /mcp__<server-name>__<prompt-name> [arguments]
        b.特征
            a.动态发现
                在以下情况下，MCP 命令自动可用：
                MCP 服务器已连接并处于活动状态
                服务器通过 MCP 协议公开提示
                连接过程中提示已成功检索
            b.参数
                MCP 提示可以接受服务器定义的参数：
                # Without arguments
                > /mcp__github__list_prs

                # With arguments
                > /mcp__github__pr_review 456
                > /mcp__jira__create_issue "Bug title" high
            c.命名约定
                服务器和提示名称已规范化
                空格和特殊字符变为下划线
                为了保持一致性，名称均采用小写
        c.管理 MCP 连接
            a.使用该/mcp命令可以
                查看所有已配置的 MCP 服务器
                检查连接状态
                使用支持 OAuth 的服务器进行身份验证
                清除身份验证令牌
                查看每个服务器可用的工具和提示

    a.设置文件
        a.说明
            该settings.json文件是我们通过分层设置来配置Claude代码的官方机制：
            用户设置在所有项目中定义~/.claude/settings.json并应用于所有项目。
            项目设置保存在您的项目目录中：
            .claude/settings.json对于签入源代码控制并与您的团队共享的设置
            .claude/settings.local.json未签入的设置，适用于个人偏好和实验。Claude Code 会.claude/settings.local.json在创建时将 git 配置为忽略。
            对于 Claude Code 的企业部署，我们还支持企业托管策略设置。这些设置优先于用户和项目设置。系统管理员可以 通过 WSL 在/Library/Application Support/ClaudeCode/policies.jsonmacOS 以及 /etc/claude-code/policies.jsonLinux 和 Windows 上部署策略。
