# 内容管理指南

本指南详细介绍如何在 TechVerse 博客系统中创建、组织和管理文章内容。

## 📝 文章创建流程

### 1. 编写 HTML 文章

在 `public/` 目录下创建文章文件。推荐的组织方式：

```
public/
├── tech/                      # 技术类文章
│   ├── frontend/             # 前端技术
│   ├── backend/              # 后端技术
│   ├── ai/                   # AI/ML
│   └── tools/                # 开发工具
├── tutorial/                 # 教程类
├── notes/                   # 学习笔记
├── thoughts/                # 随想/观点
└── resources/               # 资源整理
```

## 📂 文件命名规范

### 推荐的命名规则

- 使用小写字母和连字符
- 避免使用特殊字符和空格
- 保持简洁但有描述性

示例：
```
✅ react-hooks-guide.html
✅ nextjs-15-features.html
✅ docker-best-practices.html
✅ typescript-advanced-types.html

❌ React Hooks Guide.html
❌ nextjs 15 features.html
❌ docker@best-practices.html
❌ TypeScript's Advanced Types.html
```

### URL 路径规则

文章 URL 直接映射到文件路径：

```
文件路径: public/tech/frontend/react-hooks.html
访问URL: /article/tech/frontend/react-hooks
```

## 📋 文章元数据配置

在 `data/articles.json` 中添加文章信息：

```json
{
  "config": {
    "siteName": "TechVerse",
    "siteDescription": "探索技术的无限可能"
  },
  "articles": [
    {
      "title": "深入理解 React Hooks",
      "description": "本文将详细介绍 React Hooks 的工作原理，包括 useState、useEffect 等常用 hooks 的使用方法和最佳实践。",
      "htmlFile": "tech/frontend/react-hooks.html"
    },
    {
      "title": "Next.js 15 新特性详解",
      "description": "全面解析 Next.js 15 带来的新特性，包括 App Router 改进、性能优化和开发体验提升。",
      "htmlFile": "tech/nextjs-15-features.html"
    }
  ]
}
```

### 元数据说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | ✅ | 文章标题，显示在列表和搜索结果 |
| `description` | string | ✅ | 文章描述，建议 100-200 字 |
| `htmlFile` | string | ✅ | HTML 文件路径（相对于 public） |

## 🎨 高级内容格式

### 1. 代码高亮

使用 Prism.js 实现代码高亮：

```html
<!-- 在 head 中添加 -->
<link href="https://cdnjs.cloudflare.com/ajax/libs/prism/1.24.1/themes/prism-tomorrow.min.css" rel="stylesheet" />
<script src="https://cdnjs.cloudflare.com/ajax/libs/prism/1.24.1/prism.min.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/prism/1.24.1/components/prism-javascript.min.js"></script>

<!-- 在 body 结尾添加 -->
<script>
    Prism.highlightAll();
</script>
```

### 2. 数学公式

使用 KaTeX 渲染数学公式：

```html
<!-- 在 head 中添加 -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/katex@0.16.0/dist/katex.min.css">
<script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.0/dist/katex.min.js"></script>
<script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.0/dist/contrib/auto-render.min.js"></script>

<!-- 在 body 结尾添加 -->
<script>
    document.addEventListener("DOMContentLoaded", function() {
        renderMathInElement(document.body, {
            delimiters: [
                {left: '$$', right: '$$', display: true},
                {left: '$', right: '$', display: false}
            ]
        });
    });
</script>
```

### 3. 图表

使用 Mermaid 绘制流程图：

```html
<!-- 在 head 中添加 -->
<script src="https://cdn.jsdelivr.net/npm/mermaid@10/dist/mermaid.min.js"></script>
<script>
    mermaid.initialize({ startOnLoad: true });
</script>

<!-- 使用示例 -->
<div class="mermaid">
    graph TD;
        A[开始] --> B{条件判断};
        B -->|是| C[执行操作];
        B -->|否| D[结束];
        C --> D;
</div>
```

## 📊 内容组织策略

### 1. 系列文章

使用目录结构组织系列文章：

```
public/
└── tutorial/
    └── react-series/
        ├── 01-getting-started.html
        ├── 02-components.html
        ├── 03-state-management.html
        └── 04-deployment.html
```

在文章标题中注明顺序：

```json
{
  "title": "React 入门教程（一）：环境搭建",
  "description": "React 系列教程第一篇，介绍如何搭建 React 开发环境。",
  "htmlFile": "tutorial/react-series/01-getting-started.html"
}
```

### 2. 标签系统

虽然系统不内置标签功能，但可以在文章中手动添加：

```html
<div class="tags">
    <span class="tag">React</span>
    <span class="tag">JavaScript</span>
    <span class="tag">Frontend</span>
</div>

<style>
.tags { margin: 1rem 0; }
.tag {
    display: inline-block;
    background: #e2e8f0;
    color: #4a5568;
    padding: 0.25rem 0.75rem;
    border-radius: 1rem;
    font-size: 0.875rem;
    margin-right: 0.5rem;
}
</style>
```

### 3. 目录生成

自动生成文章目录：

```html
<nav class="toc">
    <h3>目录</h3>
    <ul id="toc-list"></ul>
</nav>

<script>
document.addEventListener('DOMContentLoaded', function() {
    const headings = document.querySelectorAll('h2, h3, h4');
    const tocList = document.getElementById('toc-list');

    headings.forEach((heading, index) => {
        heading.id = `heading-${index}`;
        const li = document.createElement('li');
        li.innerHTML = `<a href="#heading-${index}">${heading.textContent}</a>`;
        tocList.appendChild(li);
    });
});
</script>

<style>
.toc {
    background: #f7fafc;
    padding: 1.5rem;
    border-radius: 8px;
    margin-bottom: 2rem;
    position: sticky;
    top: 20px;
}
.toc ul { list-style: none; padding-left: 0; }
.toc a { color: #4a5568; text-decoration: none; }
.toc a:hover { color: #2d3748; }
</style>
```

## 🔄 内容更新流程

### 1. 修改文章

直接编辑对应的 HTML 文件，无需更新配置。

### 2. 重新排序

在 `data/articles.json` 中调整文章顺序：

```json
{
  "articles": [
    // 最新的文章放在前面
    {"title": "最新文章", ...},
    {"title": "较新文章", ...},
    {"title": "早期文章", ...}
  ]
}
```

### 3. 批量操作

使用脚本批量处理：

```javascript
// 批量更新文章发布日期
const articles = require('../data/articles.json');

articles.articles.forEach(article => {
    const stats = fs.statSync(`public/${article.htmlFile}`);
    article.date = stats.birthtime.toISOString().split('T')[0];
});

fs.writeFileSync('../data/articles.json', JSON.stringify(articles, null, 2));
```

## 📈 内容优化建议

### 1. SEO 优化

```html
<head>
    <title>文章标题 - 网站名称</title>
    <meta name="description" content="文章描述">
    <meta name="keywords" content="关键词1,关键词2">

    <!-- Open Graph -->
    <meta property="og:title" content="文章标题">
    <meta property="og:description" content="文章描述">
    <meta property="og:image" content="封面图片URL">

    <!-- Twitter Card -->
    <meta name="twitter:card" content="summary_large_image">
    <meta name="twitter:title" content="文章标题">
    <meta name="twitter:description" content="文章描述">
</head>
```

### 2. 性能优化

- 图片优化：使用 WebP 格式，添加懒加载
- CSS 优化：避免内联大量样式
- JavaScript 优化：按需加载，避免阻塞渲染

### 3. 可访问性

```html
<!-- 语义化标签 -->
<article>
    <header>
        <h1>标题</h1>
        <time datetime="2024-12-12">2024年12月12日</time>
    </header>
    <main>文章内容</main>
</article>

<!-- ARIA 标签 -->
<button aria-label="返回首页">
    <img src="back.svg" alt="">
</button>
```

## 🔍 内容管理工具推荐

### 1. 编辑器

- **VS Code**: 配置 HTML/CSS 插件
- **Typora**: 所见即所得的 Markdown 编辑器（可导出 HTML）
- **WebStorm**: 功能强大的 IDE

### 2. 辅助工具

- **Prettier**: 代码格式化
- **HTMLHint**: HTML 代码检查
- **ImageOptim**: 图片压缩工具
- **Cloudinary**: 图片托管和优化

### 3. 内容管理

- **Notion**: 草稿管理
- **GitHub**: 版本控制
- **Git LFS**: 大文件管理

## 📝 最佳实践总结

1. **保持一致性**: 使用统一的模板和样式规范
2. **优化性能**: 压缩图片，避免过度设计
3. **定期备份**: 使用 Git 管理所有内容
4. **SEO 友好**: 合理使用标题和元数据
5. **移动优先**: 确保在移动设备上的阅读体验
