## 1. VSCode插件2.0+配置

### 1.1 跳过登录配置

VSCode插件升级到2.0版本后，需要特定配置才能跳过登录：

#### 步骤1: 创建触发器文件

创建 `~/.claude/config.json` 文件：

```json
{
  "api_key": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
}
```

> ⚠️ **注意**: 这个文件是触发器，API key可以任意填写，主要作用是激活插件的环境变量读取功能。

#### 步骤2: 配置VSCode设置

在VSCode的 `settings.json` 中添加：

```json
{
  "claude-code.environmentVariables": [
    {
      "name": "ANTHROPIC_AUTH_TOKEN",
      "value": "你的实际API密钥"
    },
    {
      "name": "ANTHROPIC_BASE_URL",
      "value": "https://open.bigmodel.cn/api/anthropic"
    }
  ]
}
```

### 1.2 完整的VSCode配置示例

```json
{
  // Claude Code 插件配置
  "claude-code.environmentVariables": [
    {
      "name": "ANTHROPIC_AUTH_TOKEN",
      "value": "b04049f4fef843a3999b5fc69c7cfe5d.lDUHLQUbMv61LYE1"
    },
    {
      "name": "ANTHROPIC_BASE_URL",
      "value": "https://open.bigmodel.cn"
    },
    {
      "name": "ANTHROPIC_DEFAULT_SONNET_MODEL",
      "value": "glm-4.6"
    },

      "name": "ANTHROPIC_DEFAULT_HAIKU_MODEL",
      "value": "glm-4.5-air"
    },
    {
      "name": "ANTHROPIC_DEFAULT_OPUS_MODEL",
      "value": "glm-4.6"
    }
  ],

  // 其他VSCode配置
  "editor.formatOnSave": true,
  "files.autoSave": "afterDelay"
}
```

## 2. GLM模型配置方案

### 2.1 GLM配置参数

使用智谱GLM模型时的完整配置：

```bash
# 基础配置
export ANTHROPIC_BASE_URL="https://open.bigmodel.cn/api/anthropic"
export ANTHROPIC_AUTH_TOKEN="你的智谱API密钥"

# 模型映射（可选）
export ANTHROPIC_DEFAULT_HAIKU_MODEL="glm-4.5-air"
export ANTHROPIC_DEFAULT_SONNET_MODEL="glm-4.6"
export ANTHROPIC_DEFAULT_OPUS_MODEL="glm-4.6"
export ANTHROPIC_MODEL="GLM-4.6"
```

### 2.2 方案一：项目级配置（推荐）

在项目根目录创建 `.claude/settings.local.json`：

```json
{
  "environmentVariables": [
    {
      "name": "ANTHROPIC_AUTH_TOKEN",
      "value": "b04049f4fef843a3999b5fc69c7cfe5d.lDUHLQUbMv61LYE1"
    },
    {
      "name": "ANTHROPIC_BASE_URL",
      "value": "https://open.bigmodel.cn/api/anthropic"
    },
    {
      "name": "ANTHROPIC_DEFAULT_SONNET_MODEL",
      "value": "glm-4.6"
    },
    {
      "name": "ANTHROPIC_DEFAULT_HAIKU_MODEL",
      "value": "glm-4.5-air"
    }
  ]
}
```

**优点**:
- 优先级最高，确保生效
- 不影响其他项目
- 自动被 git 忽略，密钥安全

### 2.3 方案二：Shell环境变量

在 `~/.zshrc` 或 `~/.bashrc` 中添加：

```bash
# CC-Switch: Claude Code 配置切换
export ANTHROPIC_AUTH_TOKEN="b04049f4fef843a3999b5fc69c7cfe5d.lDUHLQUbMv61LYE1"
export ANTHROPIC_BASE_URL="https://open.bigmodel.cn/api/anthropic"
export ANTHROPIC_DEFAULT_SONNET_MODEL="glm-4.6"
export ANTHROPIC_DEFAULT_HAIKU_MODEL="glm-4.5-air"

# 可选：关闭非必要流量
export CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
```

**优点**:
- 最直接，CLI和扩展都能识别
- 配置简单，易于管理

**缺点**:
- 需要重启终端/VSCode
- 全局污染环境变量

## 3. 多环境配置管理

### 3.1 配置文件位置和优先级

Claude Code按以下优先级加载配置：

1. **命令行参数** (最高优先级)
2. **项目本地配置** `.claude/settings.local.json`
3. **项目共享配置** `.claude/settings.json`
4. **用户全局配置** `~/.claude/settings.json`
5. **企业策略配置** (最低优先级)

### 3.2 项目级配置示例

#### 共享配置 (`.claude/settings.json`)

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test:*)",
      "Read(*.md)"
    ]
  },
  "env": {
    "NODE_ENV": "development",
    "PROJECT_NAME": "my-app"
  }
}
```

#### 本地配置 (`.claude/settings.local.json`)

```json
{
  "environmentVariables": [
    {
      "name": "ANTHROPIC_AUTH_TOKEN",
      "value": "个人API密钥"
    }
  ],
  "permissions": {
    "allow": [
      "Bash(npm run dev)"
    ]
  }
}
```

### 3.3 多团队协作配置

```json
{
  "team": "frontend-team",
  "env": {
    "TEAM_SLACK_WEBHOOK": "https://hooks.slack.com/...",
    "CODE_REVIEW_ASSIGN": "@senior-dev"
  },
  "permissions": {
    "allow": [
      "Bash(git commit -m *)",
      "Bash(npm run format)"
    ],
    "deny": [
      "Bash(git push --force)",
      "Bash(rm -rf *)"
    ]
  }
}
```

## 4. 配置优先级详解

### 4.1 配置冲突处理

当多个配置源存在冲突时，Claude Code按以下顺序应用：

| 优先级 | 配置源 | 说明 |
|--------|--------|------|
| 1 | 命令行参数 | `--model`, `--api-key` 等 |
| 2 | 环境变量 | `ANTHROPIC_*` 系列变量 |
| 3 | 项目本地配置 | `.claude/settings.local.json` |
| 4 | 项目共享配置 | `.claude/settings.json` |
| 5 | 用户全局配置 | `~/.claude/settings.json` |
| 6 | 默认配置 | Claude Code内置默认值 |

### 4.2 配置继承和覆盖

```json
// 全局配置 ~/.claude/settings.json
{
  "permissions": {
    "allow": ["Read(*)"],
    "deny": ["Bash(rm *)"]
  },
  "env": {
    "EDITOR": "code"
  }
}

// 项目配置 .claude/settings.json
{
  "permissions": {
    // 继承全局allow，添加新的权限
    "allow": ["Bash(npm run *)"]
  },
  // 覆盖全局EDITOR
  "env": {
    "EDITOR": "vim"
  }
}

// 最终生效的配置
{
  "permissions": {
    "allow": ["Read(*)", "Bash(npm run *)"],
    "deny": ["Bash(rm *)"]
  },
  "env": {
    "EDITOR": "vim"  // 项目配置覆盖了全局配置
  }
}
```

## 5. 企业级部署

### 5.1 企业策略配置

企业可以通过策略文件统一管理配置：

#### macOS

```json
// /Library/Application Support/ClaudeCode/policies.json
{
  "permissions": {
    "allow": [
      "Read(*.js)",
      "Read(*.ts)",
      "Read(*.md)",
      "Bash(npm run test)",
      "Bash(npm run build)"
    ],
    "deny": [
      "Bash(curl *)",
      "Bash(wget *)",
      "Bash(rm -rf *)"
    ]
  },
  "env": {
    "ANTHROPIC_BASE_URL": "https://company-proxy.example.com/api",
    "CLAUDE_CODE_ENABLE_TELEMETRY": "0",
    "DISABLE_TELEMETRY": "1"
  },
  "cleanupPeriodDays": 7,
  "includeCoAuthoredBy": true
}
```

#### Linux

```bash
# /etc/claude-code/policies.json
sudo mkdir -p /etc/claude-code
sudo tee /etc/claude-code/policies.json << EOF
{
  "permissions": {
    "allow": ["Read(/company/project/*)"],
    "deny": ["Write(/etc/*)", "Bash(sudo *)"]
  },
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.company.com",
    "COMPANY_POLICY": "strict"
  }
}
EOF
```

### 5.2 代理和网关配置

#### HTTP代理配置

```bash
# 系统级代理
export HTTP_PROXY="http://proxy.company.com:8080"
export HTTPS_PROXY="http://proxy.company.com:8080"
export NO_PROXY="localhost,127.0.0.1,.company.com"

# Claude Code专用代理
export ANTHROPIC_BASE_URL="https://api-gateway.company.com/anthropic"
```

#### 配置文件中的代理设置

```json
{
  "env": {
    "HTTP_PROXY": "http://proxy.company.com:8080",
    "HTTPS_PROXY": "http://proxy.company.com:8080",
    "ANTHROPIC_BASE_URL": "https://api-gateway.company.com/anthropic",
    "ANTHROPIC_API_KEY_HELPER": "/usr/local/bin/company-key-helper.sh"
  }
}
```

### 5.3 API密钥管理

#### 使用密钥助手脚本

```bash
#!/bin/bash
# /usr/local/bin/company-key-helper.sh

# 从公司密钥管理系统获取临时密钥
RESPONSE=$(curl -s -X POST \
  -H "Authorization: Bearer $COMP_SERVICE_TOKEN" \
  https://keymanager.company.com/api/keys/claude)

echo $RESPONSE | jq -r '.apiKey'
```

配置使用：

```json
{
  "apiKeyHelper": "/usr/local/bin/company-key-helper.sh",
  "env": {
    "COMP_SERVICE_TOKEN": "your-service-token"
  }
}
```

### 5.4 审计和合规

#### 启用审计日志

```json
{
  "env": {
    "CLAUDE_CODE_ENABLE_AUDIT_LOG": "1",
    "AUDIT_LOG_PATH": "/var/log/claude-code/audit.log",
    "AUDIT_LOG_FORMAT": "json"
  }
}
```

#### 禁用敏感功能

```json
{
  "env": {
    "DISABLE_BUG_COMMAND": "1",
    "DISABLE_ERROR_REPORTING": "1",
    "DISABLE_TELEMETRY": "1",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1"
  }
}
```

## 最佳实践

### 1. 安全配置

- 始终使用 `settings.local.json` 存储敏感信息
- 定期轮换API密钥
- 限制权限范围，遵循最小权限原则
- 启用审计日志

### 2. 性能优化

```json
{

    "CLAUDE_CODE_MAX_OUTPUT_TOKENS": "8192",
    "MCP_TOOL_TIMEOUT": "60000"
  }

```

- 使用项目级共享配置统一团队规范
- 建立配置版本控制流程
- 定期审查和更新权限设置
