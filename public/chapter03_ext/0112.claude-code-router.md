## 1. 什么是 Claude Code Router？

Claude Code Router 是一个智能的请求路由工具，专为 Claude Code 用户设计，允许将请求灵活地路由到不同的 AI 模型和 API 端点。它提供了负载均衡、故障转移、请求定制等高级功能，让用户能够优化性能、降低成本并提高可靠性。

### 核心价值

- **智能路由**：根据不同条件智能路由请求到最适合的模型
- **负载均衡**：在多个提供商之间分配负载，提高性能
- **故障转移**：自动处理 API 故障，确保服务连续性
- **成本优化**：根据成本和性能要求选择最优路由
- **请求定制**：支持自定义请求参数和处理逻辑

### 解决的痛点

1. **单点故障**：依赖单一 API 提供商存在风险
2. **性能瓶颈**：某些提供商响应速度慢或不稳定
3. **成本控制**：不同提供商的价格差异较大
4. **功能限制**：某些模型在特定任务上表现更好
5. **复杂性管理**：手动管理多个 API 端点很繁琐

## 3. 系统架构

### 技术架构
```
Claude Code Client
    ↓
Claude Code Router
    ├── Request Receiver
    ├── Route Engine
    ├── Load Balancer
    ├── Failover Manager
    └── Analytics Engine
    ↓
Multiple AI Providers
    ├── OpenAI API
    ├── Anthropic API
    ├── OpenRouter API
    ├── Custom APIs
    └── Local Models
```

### 核心组件

#### Route Engine（路由引擎）
- **规则解析**：解析路由规则和条件
- **请求分析**：分析请求内容和类型
- **路由决策**：做出路由决策
- **策略执行**：执行路由策略

#### Load Balancer（负载均衡器）
- **分配算法**：实现各种负载均衡算法
- **健康检查**：监控提供商健康状态
- **权重管理**：管理和调整权重
- **性能监控**：监控性能指标

#### Failover Manager（故障转移管理器）
- **故障检测**：检测 API 故障
- **切换逻辑**：执行故障切换
- **恢复检测**：检测服务恢复
- **状态管理**：管理提供商状态

#### Analytics Engine（分析引擎）
- **数据收集**：收集请求和响应数据
- **统计分析**：进行统计分析
- **报告生成**：生成各类报告
- **趋势分析**：分析使用趋势

## 4. 安装指南

### 4.1 系统要求

#### 最低要求
- **操作系统**：Linux (Ubuntu 18.04+), macOS 10.15+, Windows 10+
- **处理器**：2核心 CPU
- **内存**：4GB RAM
- **存储空间**：1GB 可用空间
- **网络**：稳定的互联网连接

#### 推荐配置
- **操作系统**：Linux (Ubuntu 20.04+), macOS 12+, Windows 11+
- **处理器**：4核心或更多 CPU
- **内存**：8GB RAM 或更多
- **存储空间**：5GB 可用空间
- **网络**：高速宽带连接

#### 依赖要求
- **Node.js**：16.0.0 或更高版本
- **Python**：3.8 或更高版本（可选）
- **Docker**：20.10+（可选）

### 4.2 安装方式

#### 方式1：npm 全局安装
```bash
# 安装
npm install -g claude-code-router

# 验证安装
claude-code-router --version
```

#### 方式2：Docker 安装
```bash
# 拉取镜像
docker pull musistudio/claude-code-router

# 运行容器
docker run -d \
  --name claude-router \
  -p 8080:8080 \
  -v $(pwd)/config:/app/config \
  musistudio/claude-code-router
```

#### 方式3：源码安装
```bash
# 克隆仓库
git clone https://github.com/musistudio/claude-code-router.git
cd claude-code-router

# 安装依赖
npm install

# 构建项目
npm run build

# 启动服务
npm start
```

### 4.3 初始配置

#### 创建配置文件
```bash
# 初始化配置
claude-code-router init

# 这将在当前目录创建 config.yaml 文件
```

#### 基础配置示例
```yaml
# config.yaml
server:
  host: "0.0.0.0"
  port: 8080
  workers: 4

providers:
  openai:
    name: "OpenAI"
    api_key: "${OPENAI_API_KEY}"
    base_url: "https://api.openai.com/v1"
    models: ["gpt-4", "gpt-3.5-turbo"]
    timeout: 30000
    max_retries: 3

  anthropic:
    name: "Anthropic"
    api_key: "${ANTHROPIC_API_KEY}"
    base_url: "https://api.anthropic.com"
    models: ["claude-3-opus-20240229", "claude-3-sonnet-20240229"]
    timeout: 30000
    max_retries: 3

routing:
  default_provider: "anthropic"
  rules:
    - name: "coding_tasks"
      condition: "task_type == 'coding'"
      provider: "anthropic"
      model: "claude-3-opus-20240229"

    - name: "simple_tasks"
      condition: "complexity < 5"
      provider: "openai"
      model: "gpt-3.5-turbo"
```

## 5. 配置说明

      requests_per_minute: 3500
      tokens_per_minute: 90000

### 5.2 路由规则配置

#### 基础路由规则
```yaml
routing:
  default_provider: "anthropic"
  default_model: "claude-3-sonnet-20240229"

  rules:
    - name: "complex_coding"
      condition: |
        task_type == 'coding' and
        (file_count > 5 or line_count > 1000)
      provider: "anthropic"
      model: "claude-3-opus-20240229"
      priority: 10

    - name: "simple_coding"
      condition: |
        task_type == 'coding' and
        file_count <= 5 and line_count <= 1000
      provider: "openai"
      model: "gpt-4"
      priority: 8

    - name: "cost_optimized"
      condition: "cost_priority == 'high'"
      provider: "openai"
      model: "gpt-3.5-turbo"
      priority: 5

    - name: "speed_priority"
      condition: "speed_priority == 'high'"
      provider: "fast_provider"
      model: "fast-model"
      priority: 9
```

#### 高级路由条件
```yaml
routing:
  rules:
    - name: "time_based_routing"
      condition: |
        current_hour >= 9 and current_hour <= 17
      provider: "anthropic"
      model: "claude-3-opus-20240229"

    - name: "weekend_routing"
      condition: "is_weekend"
      provider: "openai"
      model: "gpt-4"

    - name: "large_context"
      condition: "total_tokens > 8000"
      provider: "anthropic"
      model: "claude-3-opus-20240229"

    - name: "multilingual"
      condition: "languages.length > 1"
      provider: "anthropic"
      model: "claude-3-sonnet-20240229"
```

### 5.3 负载均衡配置

#### 轮询策略
```yaml
load_balancing:
  strategy: "round_robin"
  providers:
    - name: "anthropic"
      weight: 2
      models: ["claude-3-opus-20240229"]
    - name: "openai"
      weight: 1
      models: ["gpt-4"]
    - name: "openrouter"
      weight: 1
      models: ["anthropic/claude-3-opus"]

  health_check:
    enabled: true
    interval: 60  # seconds
    timeout: 10   # seconds
    failure_threshold: 3
    recovery_threshold: 2
```

#### 加权策略
```yaml
load_balancing:
  strategy: "weighted_round_robin"
  weights:
    anthropic: 3  # 60% of requests
    openai: 2     # 40% of requests

  adaptive_weights:
    enabled: true
    metrics: ["response_time", "success_rate", "cost"]
    update_interval: 300  # seconds
```

### 5.4 故障转移配置

#### 故障转移策略
```yaml
failover:
  enabled: true
  max_retries: 3
  retry_delay: 1000  # milliseconds

  providers:
    primary: "anthropic"
    fallback:
      - provider: "openai"
        condition: "timeout || error_rate > 0.1"
      - provider: "openrouter"
        condition: "cost_budget_exceeded"

  circuit_breaker:
    enabled: true
    failure_threshold: 5
    recovery_timeout: 60000  # milliseconds
    half_open_max_calls: 3
```

## 6. 使用方法

### 6.1 基础使用

#### 启动路由服务
```bash
# 使用默认配置启动
claude-code-router

# 使用指定配置文件
claude-code-router --config /path/to/config.yaml

# 启动调试模式
claude-code-router --debug

# 指定端口
claude-code-router --port 9090
```

#### 配置 Claude Code 使用路由
```bash
# 设置环境变量
export ANTHROPIC_API_URL=http://localhost:8080/anthropic
export OPENAI_API_BASE=http://localhost:8080/openai

# 或在 Claude Code 配置中设置
```

### 6.2 API 使用

#### 发送请求到路由器
```bash
# 通过路由器发送请求
curl -X POST http://localhost:8080/v1/messages \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-api-key" \
  -d '{
    "model": "claude-3-sonnet-20240229",
    "messages": [{"role": "user", "content": "Hello!"}],
    "max_tokens": 1000
  }'
```

#### 指定路由策略
```bash
# 在请求中指定路由参数
curl -X POST http://localhost:8080/v1/messages \
  -H "Content-Type: application/json" \
  -H "X-Router-Strategy: cost_optimized" \
  -H "X-Router-Priority: high" \
  -d '{
    "model": "auto",
    "messages": [{"role": "user", "content": "Help me code"}]
  }'
```

### 6.3 监控和管理

#### 查看路由状态
```bash
# 查看所有提供商状态
curl http://localhost:8080/status

# 查看路由统计
curl http://localhost:8080/stats

# 查看健康检查状态
curl http://localhost:8080/health
```

#### 动态配置更新
```bash
# 更新路由规则
curl -X PUT http://localhost:8080/config/routing \
  -H "Content-Type: application/json" \
  -d '{
    "rules": [
      {
        "name": "new_rule",
        "condition": "task_type == 'analysis'",
        "provider": "anthropic",
        "model": "claude-3-opus-20240229"
      }
    ]
  }'
```

## 7. 故障排除

### 常见问题

#### 1. 路由器启动失败
```bash
# 检查配置文件语法
claude-code-router validate --config config.yaml

# 检查端口占用
lsof -i :8080

# 查看详细错误日志
claude-code-router --debug
```

#### 2. 提供商连接失败
```bash
# 测试提供商连接
curl -H "Authorization: Bearer your-key" https://api.anthropic.com/v1/messages

# 检查 API 密钥
claude-code-router test-provider anthropic

# 查看提供商状态
curl http://localhost:8080/providers/status
```

#### 3. 路由规则不生效
```bash
# 验证路由规则
claude-code-router validate-rules

# 查看路由决策日志
grep "routing_decision" logs/router.log

# 测试路由条件
curl -X POST http://localhost:8080/test/route \
  -d '{
    "request": {"content": "test"},
    "condition": "task_type == '\''coding'\''"
  }'
```

#### 4. 性能问题
```bash
# 查看性能指标
curl http://localhost:8080/metrics

# 分析慢请求
grep "slow_request" logs/router.log

# 优化配置
claude-code-router optimize --config config.yaml
```
