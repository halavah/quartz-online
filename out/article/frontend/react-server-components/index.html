React Server Components 实践指南

React Server Components（RSC）是 React 团队推出的革命性特性，它允许组件在服务器端渲染，带来更好的性能和开发体验。本文将深入探讨 RSC 的核心概念和实际应用。

什么是 Server Components？

Server Components 是一种特殊的 React 组件，它们在服务器上运行，而不是在客户端。这使得它们能够直接访问后端资源，同时保持组件化的开发体验。

核心优势

主要优势：
  • 零客户端 JavaScript 包大小
  • 直接访问服务器资源（数据库、文件系统等）
  • 自动代码分割
  • 更好的首屏加载性能

基本语法

Server Components 使用 'use server' 指令来标识：

// ServerComponent.server.js
'use server'

import { db } from './database'

export default async function PostList() {
  const posts = await db.posts.findMany()

  return (
    <div>
      {posts.map(post => (
        <div key={post.id}>
          <h2>{post.title}</h2>
          <p>{post.content}</p>
        </div>
      ))}
    </div>
  )
}

与 Client Components 的区别

注意： Server Components 不能使用 hooks、事件处理器或浏览器 API。需要交互的组件应该使用 Client Components。 
// ClientComponent.client.js
'use client'

import { useState, useEffect } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  )
}

最佳实践

1. 混合使用 Server 和 Client Components

在 Server Components 中嵌入 Client Components：

// PostPage.server.js
import Counter from './Counter.client'

export default function PostPage({ post }) {
  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
      <Counter />
    </article>
  )
}

2. 数据获取模式

利用 Server Components 的优势进行数据获取：

// UserProfile.server.js
async function getUser(id) {
  const res = await fetch(`https://api.example.com/users/${id}`)
  return res.json()
}

export default async function UserProfile({ userId }) {
  const user = await getUser(userId)

  return (
    <div>
      <h1>{user.name}</h1>
      <p>{user.email}</p>
    </div>
  )
}

性能优化技巧

  • 使用 Suspense 处理加载状态
  • 合理拆分组件，避免不必要的重渲染
  • 利用 Streaming SSR 提升感知性能

总结

React Server Components 代表了 React 的未来方向。通过将组件分层（Server vs Client），我们能够构建更高效、更易于维护的应用。在实际项目中，合理使用 Server Components 可以显著提升应用性能。
