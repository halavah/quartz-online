# 05. Vercel 部署详解

## 🎯 为什么选择 Vercel

### 核心优势
- **零配置部署** - 自动检测 Next.js 项目，无需复杂配置
- **全球 CDN** - 边缘网络加速，全球访问速度优化
- **自动 HTTPS** - 免费 SSL 证书，自动续期
- **Git 集成** - 推送代码自动部署
- **预览部署** - 每个 PR 自动生成预览链接
- **性能分析** - 内置 Web Vitals 监控

### 免费额度
- 100GB 带宽/月
- 100 次构建/月
- 无限静态站点
- 无限项目
- 无限团队协作

## 🚀 快速开始

### 方法一：通过 GitHub 集成（推荐）

1. **准备工作**
   ```bash
   # 确保 Git 已初始化
   git init
   git add .
   git commit -m "Initial commit"

   # 推送到 GitHub
   git remote add origin https://github.com/your-username/your-repo.git
   git push -u origin main
   ```

2. **部署步骤**
   - 访问 [vercel.com](https://vercel.com)
   - 点击 "Sign up" 使用 GitHub 账号登录
   - 点击 "New Project"
   - 选择 GitHub 仓库
   - Vercel 自动检测 Next.js 框架
   - 点击 "Deploy"

3. **完成部署**
   - 等待 1-2 分钟部署完成
   - 获得 `.vercel.app` 域名
   - 可自定义域名

### 方法二：使用 Vercel CLI

1. **安装 CLI**
   ```bash
   npm i -g vercel
   ```

2. **登录**
   ```bash
   vercel login
   # 选择 GitHub/GitLab/Email 登录方式
   ```

3. **部署**
   ```bash
   vercel --prod
   # 按提示操作，首次部署会创建项目
   ```

## ⚙️ Vercel 配置文件

虽然 Vercel 可以零配置运行，但创建 `vercel.json` 可以获得更多控制：

```json
{
  "version": 2,
  "builds": [
    {
      "src": "package.json",
      "use": "@vercel/next"
    }
  ],
  "routes": [
    {
      "src": "/article/(.*)",
      "dest": "/article/$1.html"
    },
    {
      "src": "/(.*)",
      "dest": "/$1.html"
    }
  ]
}
```

### 配置说明
- `version`: 配置文件版本（建议使用 2）
- `builds`: 构建配置，使用官方 Next.js builder
- `routes`: 路由规则，处理静态页面路由

## 🌍 环境变量配置

### 1. 在 Vercel 控制台配置

1. 进入项目设置
2. 点击 "Environment Variables"
3. 添加变量：
   ```
   NEXT_PUBLIC_SITE_URL=https://your-domain.com
   NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
   ```

### 2. 使用 `.env.local`（本地开发）

创建 `.env.local`：
```env
NEXT_PUBLIC_SITE_URL=http://localhost:3000
NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
```

### 环境变量命名规则
- `NEXT_PUBLIC_*`: 客户端可访问
- 其他变量：仅服务端可访问

## 🔧 自定义域名配置

### 步骤 1：添加域名

1. 进入项目设置
2. 点击 "Domains"
3. 输入域名（如 `blog.example.com`）
4. 点击 "Add"

### 步骤 2：配置 DNS

#### 方式一：推荐（CNAME）
```
Type: CNAME
Name: blog
Value: cname.vercel-dns.com
TTL: 300
```

#### 方式二：A 记录（根域名）
```
Type: A
Name: @
Value: 76.76.19.19
TTL: 300
```

### 步骤 3：验证

等待 DNS 传播（通常 5-30 分钟），然后：
- 检查域名是否可访问
- 验证 HTTPS 证书
- 测试所有页面路由

## 📊 性能优化设置

### 1. 构建优化

在 `next.config.ts` 中添加：
```typescript
const nextConfig = {
  // 启用压缩
  compress: true,

  // 优化图片
  images: {
    domains: ['example.com'],
    formats: ['image/webp', 'image/avif'],
  },

  // 启用实验性功能
  experimental: {
    optimizePackageImports: ['@heroicons/react'],
    scrollRestoration: true,
  },
};
```

### 2. 缓存策略

Vercel 自动缓存：
- 静态资源：1 年
- HTML 页面：根据缓存头
- API 路由：默认不缓存

### 3. 边缘函数（可选）

创建 `api/edge-function.ts`：
```typescript
export const config = {
  runtime: 'edge',
};

export default function handler(request: Request) {
  return new Response('Hello from Edge!');
}
```

## 🛠️ 常见问题解决

### 问题 1：构建失败

**原因**：Node.js 版本不匹配
```json
// 在 package.json 中指定
{
  "engines": {
    "node": ">=18.0.0"
  }
}
```

### 问题 2：路由 404

**解决方案**：确保 `next.config.ts` 正确配置：
```typescript
const nextConfig = {
  output: 'export',
  trailingSlash: true,
  images: {
    unoptimized: true
  }
};
```

### 问题 3：样式不正确

**排查步骤**：
1. 检查 Tailwind 配置
2. 确认构建命令正确
3. 查看构建日志错误

### 问题 4：环境变量不生效

**注意事项**：
- 确保变量名以 `NEXT_PUBLIC_` 开头（客户端）
- 部署后重新构建
- 检查变量值是否正确

## 💰 成本优化建议

### 免费套餐最佳实践

1. **优化图片**：使用 WebP 和 AVIF 格式
2. **代码分割**：懒加载非关键代码
3. **缓存利用**：合理设置缓存头
4. **Bundle 分析**：
   ```bash
   npm install @next/bundle-analyzer
   npm run analyze
   ```

### 升级时机

- 超过 100GB 带宽
- 需要更多构建次数
- 需要边缘函数
- 需要团队协作功能

## 📚 进阶功能

### 1. 多环境部署

使用分支管理不同环境：
- `main` → 生产环境
- `staging` → 预发布环境
- `dev` → 开发环境

### 2. 自定义 404 页面

创建 `pages/404.js`：
```typescript
export default function Custom404() {
  return <h1>404 - Page Not Found</h1>;
}
```

### 3. 中间件使用

创建 `middleware.ts`：
```typescript
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';

export function middleware(request: NextRequest) {
  // 重定向逻辑
  if (request.nextUrl.pathname === '/home') {
    return NextResponse.redirect(new URL('/', request.url));
  }
}
```

### 4. A/B 测试

使用 Vercel Edge Config：
```typescript
import { get } from '@vercel/edge-config';

export default async function handler(req) {
  const config = await get('feature-flag');
  // 根据 config 返回不同内容
}
```
