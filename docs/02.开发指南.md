# 02. 开发指南

## 📐 功能需求

### 核心功能模块

#### 1. 文章展示系统

**首页展示**：
- Hero 区域（站点标题 + 简介）
- 分类筛选按钮组
- 文章卡片网格布局（3 列）
- 分页导航（每页 6 篇文章）

**文章详情页**：
- 导航栏
- 文章头部信息（标题、描述、分类）
- HTML 内容渲染区域
- 从 `public/` 目录加载外部 HTML 文件
- 如果文件不存在，显示友好提示

#### 2. 搜索功能

**触发方式**：
- 点击导航栏搜索图标
- 弹出全屏搜索界面

**搜索特性**：
- **实时搜索**：支持搜索标题、描述、分类
- **毛玻璃效果**：半透明黑色背景 + 8px 模糊
- **渐变搜索框**：根据主题自动切换
- **结果展示**：最多显示 8 条结果，卡片式布局
- **快捷键**：ESC 关闭，自动聚焦输入框

**搜索算法**：
```typescript
// 简单的包含匹配
const query = searchQuery.toLowerCase();
return articles.filter(article =>
  article.title.toLowerCase().includes(query) ||
  article.description.toLowerCase().includes(query) ||
  article.category.toLowerCase().includes(query)
);
```

#### 3. 主题切换系统

**支持模式**：
- 暗色主题（默认）
- 明亮主题

**实现方式**：
- CSS 变量 + `data-theme` 属性
- localStorage 持久化保存
- 页面加载时自动恢复用户偏好
- 所有组件自动适配主题

**主题切换逻辑**：
```typescript
const toggleTheme = () => {
  const newTheme = theme === 'dark' ? 'light' : 'dark';
  setTheme(newTheme);
  document.documentElement.setAttribute('data-theme', newTheme);
  localStorage.setItem('theme', newTheme);
};
```

#### 4. 分类筛选系统

**功能特性**：
- 自动统计每个分类的文章数量
- 点击分类按钮筛选对应文章
- "全部"按钮显示所有文章
- 筛选后自动重置分页到第一页

**默认分类**：
- 全部
- AI工具
- 开发工具
- 技术趋势
- 前端框架

#### 5. 分页系统

**分页设置**：
- 每页显示 6 篇文章
- 可在 `app/page.tsx` 中修改 `ARTICLES_PER_PAGE` 常量

**交互体验**：
- 点击页码跳转
- 自动滚动到页面顶部
- 显示当前页码高亮

## 📝 开发注意事项

### 1. 主题适配

所有新增组件必须使用 CSS 变量，确保主题切换正常：

```css
/* ✅ 正确 */
background: var(--card-bg);
color: var(--text-color);

/* ❌ 错误 */
background: #1a1f3a;
color: #e2e8f0;
```

### 2. 响应式设计

所有布局必须考虑移动端适配：

```tsx
// 使用 Tailwind 响应式类
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3">
```

### 3. 性能优化

- 使用 `useMemo` 缓存计算结果
- 使用 `useCallback` 缓存函数引用
- 避免不必要的重渲染

```typescript
// 使用 useMemo 优化搜索性能
const searchResults = useMemo(() => {
  if (!searchQuery.trim()) return [];
  return articles.filter(/* ... */);
}, [searchQuery, articles]);
```

### 4. 静态导出限制

**不支持的特性**：
- 服务端 API 路由（`app/api/`）
- 动态路由的 `getServerSideProps`
- Image Optimization（需设置 `unoptimized: true`）
- Internationalized Routing

**支持的特性**：
- 静态路由
- 客户端数据获取
- CSS 模块
- Tailwind CSS

## 🎯 最佳实践

### 1. 组件设计

- **单一职责**：每个组件只做一件事
- **可复用性**：通过 props 提供灵活性
- **类型安全**：使用 TypeScript 定义 Props 类型

### 2. 状态管理

- **局部状态**：使用 `useState`
- **派生状态**：使用 `useMemo`
- **副作用**：使用 `useEffect`

### 3. 代码组织

```
app/
├── components/        # 可复用组件
│   ├── Navbar.tsx
│   └── Pagination.tsx
├── page.tsx          # 页面组件
└── globals.css       # 全局样式
```

### 4. 命名规范

- **组件**：PascalCase（`ArticleCard.tsx`）
- **函数**：camelCase（`handleClick`）
- **CSS 变量**：kebab-case（`--primary-color`）
- **文件名**：与导出组件同名
