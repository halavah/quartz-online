## 1. 文件引用技巧

### 1.1 基础引用方法

#### 直接路径引用

```bash
# 引用单个文件
@src/components/Button.tsx

# 引用配置文件
@package.json

# 引用文档
@README.md
```

#### 相对路径引用

```bash
# 相对于当前目录
@./utils/helper.js

# 引用上级目录
@../shared/constants.ts

# 引用子目录
@worker/src/index.ts
```

#### 项目根目录引用

```bash
# 从项目根目录开始
@src/App.js
@public/index.html
@scripts/build.sh
```

### 1.2 高级引用技巧

#### 多文件同时引用

```bash
# 空格分隔多个文件
@src/App.js @src/index.css @package.json

# 通配符引用
@src/components/*.tsx
@utils/*.js

# 类型过滤
@*.ts @*.tsx @*.js
```

#### 指定行范围

```bash
# 引用特定行
@worker/src/index.ts:100-120

# 引用函数定义
@src/utils/validation.ts:15-30

# 引用类的实现
@src/models/User.ts:50-100
```

#### 搜索和过滤引用

```bash
# 搜索特定函数
@function:deleteExpiredEmails

# 搜索特定类
@class:EmailService

# 搜索特定变量
@variable:API_BASE_URL
```

### 1.3 智能补全技巧

```bash
# 使用Tab键自动补全
@src/comp<Tab>  # 补全为 @src/components/

# 查看文件预览
@README.md<Tab>  # 显示文件内容预览

# 路径导航
@./<Tab>  # 显示当前目录文件列表
```

### 1.4 实用引用示例

```bash
# 快速查看项目配置
> 帮我分析 @package.json 和 @tsconfig.json 的配置

# 调试特定组件
> @src/components/LoginForm.tsx:50-80 这里的登录逻辑有问题

# 批量处理
> 检查 @tests/**/*.test.ts 中所有测试用例的覆盖率

# 文档生成
> 根据 @src/api/*.ts 生成API文档
```

## 2. 插件系统管理

### 2.1 添加插件市场

```bash
# 添加官方插件市场
claude plugin marketplace add https://github.com/anthropics/claude-code

# 验证市场安装
claude plugin marketplace list

# 预期输出
Configured marketplaces:
  ❯ claude-code-plugins
    Source: Git (https://github.com/anthropics/claude-code.git)
```

### 2.2 安装常用插件

#### Frontend Design 插件

```bash
# 安装前端设计插件
claude plugin install frontend-design

# 预期输出
✔ Successfully installed plugin: frontend-design@claude-code-plugins

# 查看已安装插件
claude plugin list
```

#### GitHub 集成插件

```bash
# 安装GitHub插件
claude plugin install github

# 配置GitHub token
export GITHUB_TOKEN="your-github-token"
```

#### 数据库插件

```bash
# 安装数据库连接插件
claude plugin install database

# 支持的数据库类型
# - PostgreSQL
# - MySQL
# - SQLite
# - MongoDB
```

### 2.3 插件使用示例

```bash
# 使用Frontend Design插件
> 使用frontend-design插件创建一个响应式导航栏

# 使用GitHub插件
> /mcp__github__list_prs
> /mcp__github__create_issue "Bug: 登录页面无法正常显示" high

# 使用数据库插件
> 查询users表中所有活跃用户
```

### 2.4 插件管理命令

```bash
# 查看插件详情
claude plugin info frontend-design

# 更新插件
claude plugin update frontend-design

# 卸载插件
claude plugin uninstall frontend-design

# 禁用插件（保留配置）
claude plugin disable frontend-design

# 启用插件
claude plugin enable frontend-design
```

## 3. Skills系统使用

### 3.1 创建Skills目录

```bash
# 在项目根目录创建
mkdir -p .claude/skills

# 添加到.gitignore（可选）
echo ".claude/settings.local.json" >> .gitignore
```

### 3.2 获取常用Skills

```bash
# 克隆官方Skills仓库
git clone https://github.com/anthropics/skills.git temp-skills

# 复制需要的Skills
cp -r temp-skills/artifacts-builder .claude/skills/
cp -r temp-skills/code-review .claude/skills/

# 清理临时目录
rm -rf temp-skills
```

### 3.3 使用Skills

#### Artifacts Builder Skill

```bash
# 引用并使用Skill
> 请使用 @.claude/skills/artifacts-builder/SKILL.md 创建一个whiteboard项目

# 执行后的结果
# 自动创建项目结构、配置文件和基础代码
```

#### Code Review Skill

```bash
# 代码审查
> 使用 @.claude/skills/code-review/SKILL.md 审查 @src/components/**/*.tsx

# 生成审查报告
# 包含问题列表、建议和改进方案
```

#### Documentation Generator Skill

```bash
# 生成文档
> 使用 @.claude/skills/doc-generator/SKILL.md 为 @src/api/ 生成API文档

# 输出格式
# Markdown文档，包含API说明、参数和示例
```

### 3.4 自定义Skill创建

#### 创建项目初始化Skill

```markdown
<!-- .claude/skills/project-init/SKILL.md -->
# Project Initialization Skill

## 功能
快速初始化新项目的基础结构和配置

## 使用方法
> 使用 @.claude/skills/project-init/SKILL.md 初始化一个 [项目类型] 项目

## 支持的项目类型
- react-app: React应用
- node-api: Node.js API服务
- python-app: Python应用
- go-service: Go微服务
```

#### 创建测试生成Skill

```markdown
<!-- .claude/skills/test-gen/SKILL.md -->
# Test Generation Skill

## 功能
为现有代码自动生成单元测试

## 使用方法
> 使用 @.claude/skills/test-gen/SKILL.md 为 [文件/目录] 生成测试

## 测试框架
- Jest (JavaScript/TypeScript)
- Pytest (Python)
- Go test (Go)
```

## 4. 常见工作流程

### 4.1 新项目初始化流程

```bash
# 1. 创建项目目录
mkdir my-new-project
cd my-new-project

# 2. 初始化Claude Code
claude /init

# 3. 配置项目信息
> 使用 @CLAUDE.md 设置项目规范：
> - 技术栈：React + TypeScript + Vite
> - 代码风格：ESLint + Prettier
> - 测试框架：Jest + React Testing Library

# 4. 创建基础结构
> 创建以下目录结构：
> - src/
>   - components/
>   - pages/
>   - utils/
>   - hooks/
>   - types/
> - tests/
> - docs/

# 5. 安装依赖并配置
> 根据技术栈安装必要的依赖包并配置
```

### 4.2 代码审查流程

```bash
# 1. 准备审查
> 使用 @.claude/skills/code-review/SKILL.md 设置代码审查规则

# 2. 审查特定文件
> 审查 @src/components/UserProfile.tsx 的代码质量

# 3. 批量审查
> 审查 @src/features/authentication/ 目录下所有文件的：
> - 代码规范
> - 安全性
> - 性能优化点
> - 测试覆盖率

# 4. 生成审查报告
> 生成详细的审查报告，包含：
> - 发现的问题
> - 修改建议
> - 优先级标记
```

### 4.3 Bug修复流程

```bash
# 1. 问题定位
> 分析 @logs/error.log 和 @src/components/LoginForm.tsx 查找登录失败原因

# 2. 复现问题
> 根据错误信息复现问题场景

# 3. 修复方案
> 提供修复方案并实施：
> - 修复代码
> - 添加测试用例
> - 更新文档

# 4. 验证修复
> 运行测试验证修复效果：
> npm test
> npm run build
```

### 4.4 性能优化流程

```bash
# 1. 性能分析
> 分析 @src/App.tsx 和 @src/components/ 的性能瓶颈

# 2. 识别问题
> 使用React DevTools分析：
> - 组件渲染次数
> - 内存使用
> - 依赖优化

# 3. 实施优化
> 优化建议：
> - 使用React.memo
> - 实施虚拟滚动
> - 懒加载组件
> - 优化状态管理

# 4. 性能测试
> 测试优化后的性能指标
```

## 5. 实战案例分享

### 5.1 案例1：快速重构遗留代码

```bash
# 场景：重构一个复杂的遗留组件

# 步骤1：理解现有代码
> 分析 @src/legacy/OldComponent.jsx 的功能和依赖关系

# 步骤2：设计新架构
> 设计新的组件结构：
> - 拆分为多个小组件
> - 使用现代React特性
> - 添加TypeScript类型

# 步骤3：逐步迁移
> 先创建新组件，保持接口兼容：
> @src/modern/NewComponent.tsx

# 步骤4：测试和替换
> 编写测试用例，验证功能一致性后替换
```

### 5.2 案例2：API接口文档生成

```bash
# 场景：为后端API生成完整文档

# 步骤1：分析API结构
> 扫描 @src/api/ 目录下所有API文件

# 步骤2：提取API信息
> 提取每个API的：
> - 路由路径
> - 请求方法
> - 参数说明
> - 响应格式
> - 错误处理

# 步骤3：生成文档
> 使用 @.claude/skills/doc-generator/SKILL.md 生成：
> - API概览
> - 接口详情
> - 示例代码
> - 错误码说明

# 步骤4：验证文档
> 测试文档中的示例代码确保准确性
```

### 5.3 案例3：自动化测试用例生成

```bash
# 场景：为现有代码补充测试用例

# 步骤1：分析代码覆盖率
> 运行 npm test -- --coverage 查看覆盖率报告

# 步骤2：识别未测试部分
> 找出 @src/utils/ 中未覆盖的函数

# 步骤3：生成测试用例
> 为每个未测试函数生成：
> - 正常场景测试
> - 边界值测试
> - 异常情况测试

# 步骤4：验证测试
> 运行新生成的测试确保通过
```

### 5.4 案例4：多语言项目迁移

```bash
# 场景：将JavaScript项目迁移到TypeScript

# 步骤1：项目分析
> 分析 @src/ 目录下的所有JS文件，识别：
> - 导入导出关系
> - 类型推断需求
> - 第三方库类型

# 步骤2：逐步迁移
> 按以下顺序迁移：
> 1. 工具函数 @src/utils/
> 2. 组件 @src/components/
> 3. 页面 @src/pages/
> 4. 配置文件

# 步骤3：类型定义
> 创建 @src/types/ 目录，定义通用类型

# 步骤4：配置优化
> 更新 tsconfig.json 和构建配置
```
