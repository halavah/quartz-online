## 1. 什么是 CCNexus？

CCNexus 是由 GitHub 开发者 lich0821 创建的开源智能 API 网关，专门为 Claude Code 和 Codex CLI 设计。它提供了一个统一的接口来管理和路由 AI API 请求，支持多个提供商、端点轮换、使用监控和负载均衡等功能，显著提升了 AI 编程助手的可靠性和性能。

### 核心价值

- **多平台支持**：统一管理 OpenAI、Gemini、Anthropic 等多个 AI 平台
- **端点轮换**：自动轮换 API 端点，避免单点故障
- **负载均衡**：智能分配请求到不同提供商和账户
- **使用监控**：实时监控 API 使用情况和成本
- **智能重试**：增强可靠性和容错能力
- **性能优化**：提高响应速度和用户体验

### 设计理念

CCNexus 遵循现代微服务和 API 网关的设计原则：
- **高可用性**：确保服务连续性和稳定性
- **可扩展性**：支持水平扩展和动态配置
- **安全性**：保护 API 密钥和用户数据
- **透明性**：对用户透明的请求路由和故障处理
- **可观测性**：完整的监控和日志记录

## 3. 系统架构

### 技术架构
```
Client Applications
    ↓
CCNexus Gateway
    ├── Request Router
    ├── Load Balancer
    ├── Health Monitor
    ├── Rate Limiter
    └── Analytics Engine
    ↓
Multiple API Providers
    ├── OpenAI API
    ├── Anthropic API
    ├── Gemini API
    └── Custom APIs
```

### 核心组件

#### Request Router（请求路由器）
- **请求解析**：解析和验证 API 请求
- **路由决策**：根据配置决定请求路由
- **协议转换**：在不同 API 协议间转换
- **响应处理**：处理和格式化 API 响应

#### Load Balancer（负载均衡器）
- **分配算法**：实现多种负载均衡算法
- **权重管理**：动态调整端点权重
- **健康检查**：定期检查端点健康状态
- **故障检测**：检测和处理端点故障

#### Health Monitor（健康监控器）
- **状态监控**：实时监控端点状态
- **性能监控**：监控端点性能指标
- **告警系统**：异常情况的告警通知
- **统计收集**：收集和分析统计数据

#### Rate Limiter（请求限制器）
- **请求限制**：基于用户和端点的请求限制
- **配额管理**：API 使用配额的管理
- **优先级处理**：基于优先级的请求处理
- **流量控制**：精确的流量控制机制

## 4. 安装指南

### 4.1 系统要求

#### 最低要求
- **操作系统**：Linux (Ubuntu 18.04+), macOS 10.15+, Windows 10+
- **处理器**：2核心 CPU
- **内存**：4GB RAM
- **存储空间**：1GB 可用空间
- **网络**：稳定的互联网连接

#### 推荐配置
- **操作系统**：Linux (Ubuntu 20.04+), macOS 12+, Windows 11+
- **处理器**：4核心或更多 CPU
- **内存**：8GB RAM 或更多
- **存储空间**：5GB 可用空间
- **网络**：高速宽带连接

#### 依赖服务
- **Go**：1.19+（如果从源码编译）
- **Docker**：20.10+（可选，用于容器化部署）
- **Redis**：6.0+（可选，用于缓存和会话存储）

### 4.2 安装方式

#### 方式1：预编译二进制文件
```bash
# Linux/macOS 下载
wget https://github.com/lich0821/ccNexus/releases/latest/download/ccnexus-linux-amd64.tar.gz
tar -xzf ccnexus-linux-amd64.tar.gz

# Windows 下载
# 下载 ccnexus-windows-amd64.zip 并解压

# 移动到系统路径
sudo mv ccnexus /usr/local/bin/

# 验证安装
ccnexus --version
```

#### 方式2：源码编译
```bash
# 克隆仓库
git clone https://github.com/lich0821/ccNexus.git
cd ccNexus

# 编译
go build -o ccnexus cmd/ccnexus/main.go

# 安装到系统路径
sudo mv ccnexus /usr/local/bin/

# 验证安装
ccnexus --version
```

#### 方式3：Docker 部署
```bash
# 拉取镜像
docker pull lich0821/ccnexus

# 运行容器
docker run -d \
  --name ccnexus \
  -p 8080:8080 \
  -v $(pwd)/config:/app/config \
  lich0821/ccnexus

# 验证部署
curl http://localhost:8080/health
```

#### 方式4：Go 安装
```bash
# 安装最新版本
go install github.com/lich0821/ccNexus@latest

# 验证安装
ccnexus --version
```

```yaml
# config.yaml
server:
  host: "0.0.0.0"
  port: 8080
  workers: 4

providers:
  openai:
    name: "OpenAI"
    base_url: "https://api.openai.com/v1"
    api_key: "${OPENAI_API_KEY}"
    models: ["gpt-4", "gpt-3.5-turbo"]
    timeout: 30000
    weight: 2

  anthropic:
    name: "Anthropic"
    base_url: "https://api.anthropic.com"
    api_key: "${ANTHROPIC_API_KEY}"
    models: ["claude-3-opus-20240229", "claude-3-sonnet-20240229"]
    timeout: 30000
    weight: 3

  gemini:
    name: "Google Gemini"
    base_url: "https://generativelanguage.googleapis.com/v1"
    api_key: "${GEMINI_API_KEY}"
    models: ["gemini-pro"]
    timeout: 30000
    weight: 1

routing:
  default_strategy: "weighted_round_robin"
  health_check_interval: 30
  max_retries: 3
  timeout: 60000

monitoring:
  enabled: true
  metrics_interval: 60
  log_level: "info"
  export_prometheus: true
```

## 5. 配置说明

### 5.1 提供商配置

#### OpenAI 配置
```yaml
providers:
  openai:
    name: "OpenAI"
    base_url: "https://api.openai.com/v1"
    api_key: "sk-your-key"
    models:
      - name: "gpt-4"
        max_tokens: 8192
        cost_per_input: 0.03
        cost_per_output: 0.06
      - name: "gpt-3.5-turbo"
        max_tokens: 4096
        cost_per_input: 0.0015
        cost_per_output: 0.002

    rate_limit:
      requests_per_minute: 3500
      tokens_per_minute: 90000

    health_check:
      endpoint: "/models"
      interval: 30
      timeout: 10

    retry_policy:
      max_retries: 3
      backoff: "exponential"
      max_delay: 60000
```

#### 自定义提供商配置
```yaml
providers:
  custom_provider:
    name: "Custom LLM Provider"
    type: "openai_compatible"
    base_url: "https://your-provider.com/v1"
    api_key: "your-key"
    models:
      - name: "custom-model"
        max_tokens: 4096
        cost_per_input: 0.01
        cost_per_output: 0.02

    headers:
      "X-Custom-Header": "value"
      "Authorization": "Bearer your-token"

    ssl_verify: false
    timeout: 60000
```

### 5.2 路由策略配置

#### 负载均衡策略
```yaml
routing:
  strategy: "weighted_round_robin"  # round_robin, weighted_round_robin, least_connections, random

  weights:
    openai: 2
    anthropic: 3
    gemini: 1

  health_check:
    enabled: true
    interval: 30
    timeout: 10
    failure_threshold: 3
    recovery_threshold: 2

  circuit_breaker:
    enabled: true
    failure_threshold: 5
    recovery_timeout: 60000
    half_open_max_calls: 3
```

#### 请求路由规则
```yaml
routing:
  rules:
    - name: "coding_tasks"
      condition: "task_type == 'coding'"
      providers: ["anthropic", "openai"]
      models: ["claude-3-opus-20240229", "gpt-4"]
      strategy: "weighted_round_robin"

    - name: "simple_tasks"
      condition: "task_complexity < 5"
      providers: ["openai", "gemini"]
      models: ["gpt-3.5-turbo", "gemini-pro"]
      strategy: "least_cost"

    - name: "high_priority"
      condition: "priority == 'high'"
      providers: ["anthropic"]
      models: ["claude-3-opus-20240229"]
      strategy: "fastest"
```

# 查看提供商状态

# 查看路由状态

### 6.2 API 使用

#### 代理请求到不同提供商
```bash
# OpenAI 请求
curl -X POST http://localhost:8080/openai/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer any-key" \
  -d '{
    "model": "gpt-4",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'

# Anthropic 请求
curl -X POST http://localhost:8080/anthropic/v1/messages \
  -H "Content-Type: application/json" \
  -H "x-api-key: any-key" \
  -d '{
    "model": "claude-3-sonnet-20240229",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

#### 使用特定路由策略
```bash
# 指定路由策略
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "X-CCNexus-Strategy: weighted_round_robin" \
  -H "X-CCNexus-Priority: high" \
  -d '{
    "model": "auto",
    "messages": [{"role": "user", "content": "Help me code"}]
  }'
```
