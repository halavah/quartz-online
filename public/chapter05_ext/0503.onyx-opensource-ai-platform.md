## 1. 项目概述

Onyx 是一个功能强大的企业级开源AI平台，旨在为组织提供完整的AI解决方案。通过支持多种AI模型、知识源集成和自定义工作流，Onyx帮助企业构建智能化的知识管理和决策支持系统。

### 🎯 设计目标

- **开源可控** - 完全开源，企业可自行部署和控制
- **企业级特性** - 满足企业安全、权限、合规要求
- **高扩展性** - 支持多种AI模型和知识源
- **用户友好** - 精美的UI界面，降低使用门槛

## 2. 核心特性

### ✨ 主要功能

- **🤖 自定义代理** - 创建专门的AI代理处理特定任务
- **🔍 RAG（检索增强生成）** - 基于企业知识库的智能问答
- **🌐 Web 搜索** - 集成实时网络搜索能力
- **📊 深度研究** - 多步骤、多源信息的深度分析
- **⚡ 动作 & MCP** - 支持Model Context Protocol，扩展AI能力
- **💻 代码解释器** - 安全的代码执行环境
- **🔗 40+ 知识源连接器** - 支持各种数据源集成
- **🔒 企业级安全** - 完整的权限管理和安全控制
- **🎨 精美 UI** - 现代化的用户界面设计


#### 1. 克隆项目

## 4. 功能详解

### 🤖 自定义代理

Onyx 支持创建专门的AI代理来处理特定任务和工作流

#### 代理类型
- **知识管理代理** - 专门处理文档检索和知识问答
- **研究助手代理** - 深度研究和分析任务
- **代码助手代理** - 代码生成、解释和调试
- **客服代理** - 基于知识库的客户支持

#### 代理配置示例
```yaml
# 知识管理代理配置
name: "Knowledge Assistant"
description: "专门处理企业知识库查询"
model: "gpt-4"
temperature: 0.1
tools:
  - rag_search
  - document_retrieval
  - web_search
prompts:
  system: "你是一个专业的知识管理助手，基于提供的文档准确回答用户问题"
```

### 🔍 RAG（检索增强生成）

#### 核心能力
- **智能检索** - 基于语义相似度的文档检索
- **上下文增强** - 将检索到的信息作为上下文提供给AI
- **多模态支持** - 支持文本、图片、表格等多种格式
- **实时更新** - 知识库实时更新，AI可以获取最新信息

#### 工作流程
```mermaid
graph LR
    A[用户查询] --> B[向量化检索]
    B --> C[相关文档检索]
    C --> D[上下文构建]
    D --> E[AI生成回答]
    E --> F[结果返回]
```

### 🌐 Web 搜索

#### 搜索引擎集成
- **Google Search** - 通过 Google Search API
- **Bing Search** - Microsoft Bing 搜索
- **Serper** - 高性能搜索API
- **Tavily** - AI优化搜索引擎

#### 搜索功能
- **实时信息** - 获取最新的网络信息
- **多语言支持** - 支持多种语言的搜索
- **搜索过滤** - 可配置搜索来源和时间范围
- **结果摘要** - AI自动总结搜索结果

### 📊 深度研究

#### 多步骤研究
- **信息收集** - 从多个来源收集信息
- **交叉验证** - 验证信息的准确性
- **趋势分析** - 分析数据趋势和模式
- **报告生成** - 自动生成研究报告

#### 研究模板
```yaml
# 市场研究模板
name: "Market Research"
steps:
  - type: web_search
    query: "市场趋势分析"
  - type: data_analysis
    data_source: "market_data.csv"
  - type: report_generation
    format: "markdown"
```

### 💻 代码解释器

#### 安全执行环境

- **时间限制** - 防止无限循环和长时间运行

#### 支持语言
- **Python** - 数据分析、机器学习、科学计算
- **JavaScript** - Web开发、数据处理

## 6. 代理配置

### 🤖 创建自定义代理

#### 基础配置
```yaml
# 代理配置文件
agent:
  name: "Research Assistant"
  description: "专门处理深度研究任务"

  # AI模型配置
  model:
    provider: "openai"
    model: "gpt-4"
    temperature: 0.2
    max_tokens: 4000

  # 工具配置
  tools:
    - name: "web_search"
      type: "search"
      config:
        engines: ["google", "serper"]
        max_results: 10

    - name: "document_retrieval"
      type: "rag"
      config:
        vector_store: "qdrant"
        top_k: 5
        threshold: 0.7

    - name: "code_interpreter"
      type: "code"
      config:
        languages: ["python", "sql"]
        timeout: 300

  # 系统提示
  system_prompt: |
    你是一个专业的研究助手。在回答问题时：
    1. 首先搜索最新的相关信息
    2. 查阅相关的内部文档
    3. 进行数据分析和验证
    4. 提供详细的研究报告
    5. 注明信息来源和时间
```

## 7. 企业级部署

### 🏢 生产环境部署

#### 高可用架构
```yaml
# docker-compose.prod.yml
version: '3.8'
services:
  onyx-web:
    image: onyxproject/onyx:latest
    replicas: 3
    deploy:
      resources:
        limits:
          cpus: '1.0'
          memory: 2G
    environment:
      - NODE_ENV=production
      - DATABASE_URL=${DATABASE_URL}

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./ssl:/etc/ssl
```

#### 负载均衡配置
```nginx
# nginx.conf
upstream onyx_backend {
    server onyx-web-1:3000;
    server onyx-web-2:3000;
    server onyx-web-3:3000;
}

server {
    listen 443 ssl;
    server_name onyx.company.com;

    ssl_certificate /etc/ssl/cert.pem;
    ssl_certificate_key /etc/ssl/key.pem;

    location / {
        proxy_pass http://onyx_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
