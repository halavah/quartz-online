# 03. 内容管理

## 📝 文章创建流程

### 步骤 1: 创建 HTML 文章文件

在 `public/` 目录下创建您的文章文件。推荐的目录组织方式：

```
public/
├── chapter01/             # 第一章节
│   ├── 01.article-name.html
│   ├── 02.another-article.html
│   └── ...
├── chapter02/             # 第二章节
├── chapter03/             # 第三章节
└── ...
```

## 🗂️ 文件命名规范

### 推荐命名格式

使用**章节编号 + 顺序编号 + 文章名称**的格式：

```
✅ chapter01/01.react-hooks-guide.html
✅ chapter02/03.nextjs-deployment.html
✅ chapter03/05.docker-best-practices.html

❌ React Hooks Guide.html
❌ nextjs 15 features.html
❌ chapter01/React教程.html
```

**命名原则**：
- 使用小写字母和连字符
- 避免使用中文、空格、特殊字符
- 保持简洁但有描述性
- 使用章节编号便于管理

### URL 路径映射

文章 URL 直接映射到文件路径（移除 `.html` 后缀）：

```
文件路径: public/chapter01/01.react-hooks.html
访问 URL: /article/chapter01/01.react-hooks
```

## 📊 分类管理

### 可用分类

系统预设了以下分类（在 `app/page.tsx` 中定义）：

```typescript
const categories = ['全部', 'AI工具', '开发工具', '技术趋势', '前端框架'];
```

### 添加新分类

**步骤 1**: 编辑 `app/page.tsx`，添加新分类：

```typescript
const categories = [
  '全部',
  'AI工具',
  '开发工具',
  '技术趋势',
  '前端框架',
  '后端技术',     // 新增
  '云原生'        // 新增
];
```

**步骤 2**: 在 `articles.json` 中使用新分类：

```json
{
  "title": "Kubernetes 入门指南",
  "description": "从零开始学习 Kubernetes...",
  "htmlFile": "chapter05/01.k8s-guide.html",
  "category": "云原生"
}
```

### 分类最佳实践

- **数量控制**：建议 4-8 个分类
- **命名简洁**：2-4 个字为宜
- **互斥性**：避免分类重叠
- **覆盖性**：确保所有文章都有合适分类

## ⚙️ 站点配置

### 修改站点信息

编辑 `data/articles.json` 中的 `config` 部分：

```json
{
  "config": {
    "siteName": "我的技术博客",
    "siteDescription": "分享技术心得，记录成长历程",
    "githubUrl": "https://github.com/your-username"
  }
}
```

**配置字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `siteName` | string | ✅ | 站点名称，显示在导航栏和页面标题 |
| `siteDescription` | string | ✅ | 站点描述，显示在首页 Hero 区域 |

## 🔄 内容更新流程

### 修改已发布文章

1. **编辑 HTML 文件**：直接修改 `public/` 目录下的文章文件
2. **更新配置**（可选）：如果修改了标题或描述，同步更新 `articles.json`
3. **重新构建**：
   ```bash
   npm run build
   ```

### 删除文章

1. **从配置中移除**：编辑 `articles.json`，删除对应文章条目
2. **删除文件**（可选）：删除 `public/` 目录下的 HTML 文件
3. **重新构建**

### 批量管理

使用脚本批量处理文章（示例）：

```javascript
// 批量更新分类
const fs = require('fs');
const articles = require('./data/articles.json');

// 将所有 "开发工具" 分类改为 "工具推荐"
articles.articles.forEach(article => {
  if (article.category === '开发工具') {
    article.category = '工具推荐';
  }
});

// 保存修改
fs.writeFileSync(
  './data/articles.json',
  JSON.stringify(articles, null, 2)
);
```

## 📈 内容优化建议

### 1. SEO 优化

在 HTML 文件中添加元数据：

```html
<head>
    <title>文章标题 - Halavah's Tech</title>
    <meta name="description" content="文章简介描述">
    <meta name="keywords" content="JavaScript, 异步编程, Promise">

    <!-- Open Graph（社交分享） -->
    <meta property="og:title" content="文章标题">
    <meta property="og:description" content="文章简介">
    <meta property="og:type" content="article">
    <meta property="og:url" content="https://yourdomain.com/article/...">
    <meta property="og:image" content="https://yourdomain.com/cover.jpg">

    <!-- Twitter Card -->
    <meta name="twitter:card" content="summary_large_image">
    <meta name="twitter:title" content="文章标题">
    <meta name="twitter:description" content="文章简介">
    <meta name="twitter:image" content="https://yourdomain.com/cover.jpg">
</head>
```

### 2. 性能优化

**图片优化**：
- 使用 WebP 格式
- 添加懒加载：`<img loading="lazy" src="...">`
- 压缩图片文件

**CSS 优化**：
- 避免内联过多样式
- 使用外部 CSS 文件（可缓存）

**JavaScript 优化**：
- 脚本放在 `</body>` 前
- 使用 `defer` 或 `async` 属性

### 3. 可访问性

使用语义化 HTML：

```html
<article>
    <header>
        <h1>文章标题</h1>
        <time datetime="2024-12-12">2024年12月12日</time>
    </header>

    <main>
        <section>
            <h2>章节标题</h2>
            <p>章节内容...</p>
        </section>
    </main>

    <footer>
        <p>作者：Halavah</p>
    </footer>
</article>
```
