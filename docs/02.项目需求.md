# 项目需求文档

## 📋 项目概述

创建一个现代化、高档次的技术博客系统，支持外部 HTML 文章挂载，配置简单，易于维护。

---

## 🎯 核心需求

### 1. 技术栈要求

- **框架**: Next.js 15+ (App Router)
- **样式**: Tailwind CSS
- **语言**: TypeScript
- **部署**: 支持静态导出 (Static Site Generation)

### 2. 设计风格要求

**必须满足**:
- ✅ **高档次、现代化**的设计风格
- ✅ 使用**内部 UI 工具**（如 Tailwind、shadcn/ui 等）
- ✅ **简洁、优雅**的视觉呈现
- ✅ **响应式设计**，完美适配桌面和移动端

**设计参考**:
- 极简主义风格
- 专业的配色方案
- 良好的视觉层次
- 舒适的阅读体验

### 3. 文章系统要求

**核心功能**:
1. **外部 HTML 文章挂载**
   - 文章内容存储为独立的 HTML 文件
   - 放置在 `public/articles/` 目录
   - 支持完整的 HTML 文档或 HTML 片段

2. **JSON 配置管理**
   - 所有文章元数据通过外置 JSON 文件配置
   - 配置文件位置: `data/articles.json`
   - 易于编辑和维护

3. **简单的写作流程**
   ```
   1. 编写 HTML 文章 → 保存到 public/articles/
   2. 更新 JSON 配置文件
   3. 完成！
   ```

## 📐 功能需求

### 页面结构

#### 1. 首页 (`/`)

**布局要求**:
- Hero 区域（网站标题 + 简介）
- 精选文章区域（展示重点推荐的文章）
- 最近文章列表（展示所有其他文章）
- 页脚

**展示内容**:
- 文章标题
- 文章简介/描述
- 文章标签
- 作者信息
- 发布日期
- 预计阅读时长

**交互要求**:
- 点击文章卡片跳转到文章详情页
- 悬停效果（hover）提升交互体验

#### 2. 文章详情页 (`/article/[id]`)

**布局要求**:
- 顶部导航（返回首页按钮）
- 文章头部信息区
  - 标题
  - 描述
  - 标签
  - 作者、日期、阅读时长
- 文章内容区（加载并渲染外部 HTML）
- 页脚

**功能要求**:
- 从 `public/articles/` 加载对应的 HTML 文件
- 渲染 HTML 内容（使用 `dangerouslySetInnerHTML` 或其他安全方式）
- 如果 HTML 文件不存在，显示友好的占位信息

## 🗂️ 数据结构

### JSON 配置文件格式

**文件位置**: `data/articles.json`

```json
{
  "config": {
    "siteName": "网站名称",
    "siteDescription": "网站描述",
    "author": "默认作者名"
  },
  "articles": [
    {
      "id": "article-url-slug",
      "title": "文章标题",
      "description": "文章简介描述",
      "htmlFile": "article-filename.html",
      "author": "作者名",
      "date": "2024-12-11",
      "readTime": "8 分钟",
      "tags": ["标签1", "标签2", "标签3"],
      "featured": true,
      "cover": "封面图片URL（可选）"
    }
  ]
}
```

**字段说明**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 文章唯一标识，用于 URL 路径 |
| `title` | string | ✅ | 文章标题 |
| `description` | string | ✅ | 文章简介，显示在列表和详情页 |
| `htmlFile` | string | ⚠️ | HTML 文件名（相对于 `public/articles/`），如果为空则显示占位页 |
| `author` | string | ✅ | 作者名称 |
| `date` | string | ✅ | 发布日期（YYYY-MM-DD 格式） |
| `readTime` | string | ✅ | 阅读时长（如 "8 分钟"） |
| `tags` | array | ✅ | 标签数组 |
| `featured` | boolean | ✅ | 是否为精选文章（true 则显示在精选区域） |
| `cover` | string | ❌ | 封面图片 URL（暂时可选） |

## 🎨 UI/UX 要求

### 设计原则

1. **简洁优先** - 避免过度设计，保持页面清爽
2. **内容为王** - 突出文章内容，减少干扰元素
3. **视觉层次** - 清晰的标题层级和信息架构
4. **响应式** - 移动端和桌面端都有良好体验
5. **高档次感** - 专业的配色、字体、间距

### 配色建议

- **主色调**: 可以使用灰色系、蓝色系等专业色调
- **强调色**: 用于交互元素（按钮、链接、hover 状态）
- **文字**: 高对比度，易读性强
- **背景**: 浅色渐变或纯色，不要太花哨

### 字体和排版

- 中文字体: 系统默认（苹方、微软雅黑等）
- 英文字体: 现代无衬线字体（如 Inter, Geist）
- 标题: 大号、粗体、紧凑间距
- 正文: 适中大小、舒适行高（1.6-1.8）

### 间距和布局

- 使用统一的间距系统（如 Tailwind 的 spacing scale）
- 合理的留白，不要让页面看起来拥挤
- 卡片使用圆角、阴影提升层次感
- 网格布局保持整齐对齐

## 🚀 技术要求

### 1. 项目结构

```
project/
├── app/
│   ├── page.tsx              # 首页
│   ├── article/
│   │   └── [id]/
│   │       └── page.tsx      # 文章详情页
│   ├── layout.tsx            # 全局布局
│   └── globals.css           # 全局样式
│
├── data/
│   └── articles.json         # 文章配置
│
├── public/
│   └── articles/             # HTML 文章文件
│       ├── article-1.html
│       ├── article-2.html
│       └── ...
│
├── components/               # 可选：组件目录
│
├── next.config.ts            # Next.js 配置
├── tailwind.config.ts        # Tailwind 配置
└── package.json
```

### 2. Next.js 配置

**必须支持静态导出**:

```typescript
// next.config.ts
const nextConfig = {
  output: 'export',           // 启用静态导出
  images: {
    unoptimized: true         // 静态导出时禁用图片优化
  },
  trailingSlash: true,        // URL 末尾添加斜杠
};
```

### 3. 文章渲染

**HTML 文件加载方式**:

```typescript
// 服务端加载 HTML 文件
import { readFile } from 'fs/promises';
import { join } from 'path';

const htmlPath = join(process.cwd(), 'public', 'articles', article.htmlFile);
const htmlContent = await readFile(htmlPath, 'utf-8');

// 渲染 HTML
<div dangerouslySetInnerHTML={{ __html: htmlContent }} />
```

**样式处理**:
- 使用 Tailwind Typography (`@tailwindcss/typography`)
- 为文章内容区域添加 `prose` 类
- 支持 HTML 文件内的自定义 `<style>` 标签

### 4. 路由和 URL

**首页**: `/` 或 `/index.html`

**文章详情**: `/article/{id}` 或 `/article/{id}/index.html`

例如:
- JSON: `"id": "nextjs-15-features"`
- 开发环境: `http://localhost:3000/article/nextjs-15-features`
- 生产环境: `https://yourdomain.com/article/nextjs-15-features/`
