## 🗂️ 文件命名规范

### 推荐命名格式

使用**章节编号 + 顺序编号 + 文章名称**的格式：

```
✅ chapter01/01.react-hooks-guide.html
✅ chapter02/03.nextjs-deployment.html
✅ chapter03/05.docker-best-practices.html

❌ React Hooks Guide.html
❌ nextjs 15 features.html
❌ chapter01/React教程.html
```

**命名原则**：
- 使用小写字母和连字符
- 避免使用中文、空格、特殊字符
- 保持简洁但有描述性
- 使用章节编号便于管理

### URL 路径映射

文章 URL 直接映射到文件路径（移除 `.html` 后缀）：

```
文件路径: public/chapter01/01.react-hooks.html
访问 URL: /article/chapter01/01.react-hooks
```

## 📊 分类管理

### 动态分类系统

系统不再硬编码分类，而是自动从 `articles.json` 中提取所有唯一分类：

```typescript
// 在 app/page.tsx 中自动生成分类
const categories = useMemo(() => {
  const uniqueCategories = new Set<string>();
  articles.forEach((article: any) => {
    const cat = article.category || '开发工具';
    uniqueCategories.add(cat);
  });
  return ['全部', ...Array.from(uniqueCategories).sort()];
}, [articles]);
```

### 分类使用指南

**添加新分类**：
只需在 `articles.json` 中使用新的分类名称，系统会自动识别：

```json
{
  "title": "Kubernetes 入门指南",
  "description": "从零开始学习 Kubernetes...",
  "htmlFile": "chapter05/01.k8s-guide.html",
  "category": "云原生"  // 新分类，系统会自动添加
}
```

**分类最佳实践**：
- **数量控制**：建议 4-8 个分类
- **命名简洁**：2-4 个字为宜
- **互斥性**：避免分类重叠
- **覆盖性**：确保所有文章都有合适分类
- **一致性**：相同主题的文章使用相同分类名称

| 字段 | 类型 | 必填 | 说明 |

## 🔄 内容更新流程


3. **重新构建**

### 批量管理

使用脚本批量处理文章（示例）：

```javascript
// 批量更新分类
const fs = require('fs');
const articles = require('./data/articles.json');

// 将所有 "开发工具" 分类改为 "工具推荐"
articles.articles.forEach(article => {
  if (article.category === '开发工具') {
    article.category = '工具推荐';
  }
});

// 保存修改
fs.writeFileSync(
  './data/articles.json',
  JSON.stringify(articles, null, 2)
);
```

```html
    <meta name="twitter:card" content="summary_large_image">
    <meta name="twitter:title" content="文章标题">
    <meta name="twitter:description" content="文章简介">
    <meta name="twitter:image" content="https://yourdomain.com/cover.jpg">
```

### 2. 性能优化

**图片优化**：
- 使用 WebP 格式
- 添加懒加载：`<img loading="lazy" src="...">`
- 压缩图片文件

**CSS 优化**：
- 避免内联过多样式
- 使用外部 CSS 文件（可缓存）

**JavaScript 优化**：
- 脚本放在 `</body>` 前
- 使用 `defer` 或 `async` 属性

### 3. 可访问性

使用语义化 HTML：

```html
            <p>章节内容...</p>
```
