新特性代码示例

通过实际代码了解 Next.js 15 的改进

新的缓存配置
typescript
// app/page.tsx
export const revalidate = 3600 // 每小时重新验证
export const dynamic = 'force-static' // 静态生成

// 动态路由
export const dynamicParams = true // 允许动态参数

// 数据获取
async function getPosts() {
  // Next.js 15 中 fetch 默认不缓存
  const res = await fetch('https://api.example.com/posts', {
    cache: 'force-cache', // 明确指定缓存
    next: {
      revalidate: 3600,
      tags: ['posts']
    }
  })

  return res.json()
}

export default async function Page() {
  const posts = await getPosts()

  return (
    <div>
      {posts.map(post => (
        &PostCard key={post.id} post={post} />
      ))}
    </div>
  )
}

Partial Prerendering
typescript
// app/dashboard/@user/[id]/page.tsx
import { Suspense } from 'react'
import UserProfile from '@/components/UserProfile'
import UserActivity from '@/components/UserActivity'

export default async function DashboardPage({ params }) {
  // 静态部分：预渲染
  const staticData = await getStaticDashboardData()

  return (
    <div>
      <h1>Dashboard</h1>

      {/* 静态内容 */}
      <nav>
        <Link href="/dashboard/settings">Settings</Link>
        <Link href="/dashboard/profile">Profile</Link>
      </nav>

      {/* 动态部分：流式渲染 */}
      <Suspense fallback={<Loading />}>
        <UserProfile userId={params.id} />
      </Suspense>

      <Suspense fallback={<Loading />}>
        <UserActivity userId={params.id} />
      </Suspense>
    </div>
  )
}

// 静态数据获取
async function getStaticDashboardData() {
  // 这些数据会被预渲染
  return {
    navigation: await getNavigationItems(),
    userStats: await getUserStats(params.id)
  }
}

迁移到 Next.js 15

从 Next.js 14 平滑升级到 15 的步骤指南

⚠️ 重要提示：

Next.js 15 包含一些重大变更，请仔细阅读迁移指南并测试所有功能。

  1. 1. 更新依赖

    更新到 Next.js 15 和 React 18 

    npm install next@15 react@18 react-dom@18

  2. 2. 更新缓存策略

    检查所有 fetch 调用，明确指定缓存行为 

    // 之前
fetch('/api/data')

// 之后
fetch('/api/data', { cache: 'force-cache' })

  3. 3. 更新路由处理程序

    GET 路由处理程序现在需要明确的缓存配置 

    export async function GET() {
  const data = await fetchData()

  return NextResponse.json(data, {
    headers: {
      'Cache-Control': 'public, s-maxage=60, stale-while-revalidate=30'
    }
  })
}

  4. 4. 启用 Turbopack（可选）

    在开发环境中启用 Turbopack 以获得更好的性能 

    "scripts": {
  "dev": "next dev --turbo"
}

  5. 5. 测试应用

    全面测试应用的各个方面，特别关注：

    • 数据获取和缓存行为
    • 路由转换
    • Server Actions
    • 静态生成页面
💡 迁移提示：
  • 先在开发环境测试所有功能
  • 使用 Vercel 的 Speed Insights 监控性能
  • 逐步迁移，可以先启用部分新特性
  • 查看官方迁移文档获取最新信息

最佳实践建议

充分利用 Next.js 15 新特性的建议

🚀 使用 Turbopack

在开发环境中启用 Turbopack，享受极速的开发体验。对于大型项目，考虑在生产构建中也使用它。

💾 智能缓存

根据数据特性选择合适的缓存策略。静态内容使用长时间缓存，动态内容使用短时间或无缓存。

🔄 Partial Prerendering

将页面拆分为静态和动态部分，使用 Partial Prerendering 获得最佳性能。

📊 性能监控

使用 Next.js 内置的性能监控工具，持续跟踪和优化应用性能。

🎯 Server Components

充分利用 Server Components 减少客户端 JavaScript，提升首屏加载速度。

🔧 代码分割

使用动态导入和 React.lazy 实现更细粒度的代码分割，减少初始包大小。
