Claude Code 原生安装完全指南

使用 Bun 编译的原生版本，无需 Node.js 依赖，启动速度提升 4 倍的全新安装体验

原生安装概述

Claude Code 原生版本是使用 Bun 编译器将整个应用打包成单一可执行文件的全新安装方式。它彻底摆脱了对 Node.js 运行时的依赖，提供了更快、更轻量的使用体验。

核心优势

  • 零依赖：无需安装 Node.js 或任何运行时环境
  • 极速启动：启动时间从 2-3 秒缩短至 0.5-0.7 秒
  • 资源优化：内存占用减少 30-50%，体积缩小 90% 以上
  • 部署简单：单文件部署，特别适合服务器和 CI/CD 环境

系统要求

支持的操作系统

  • Windows：Windows 10 1903 或更高版本（x64 架构）
  • macOS：macOS 11.0 (Big Sur) 或更高版本（Intel 或 Apple Silicon）
  • Linux：支持 glibc 2.17+ 的主流发行版（x64 架构）
    • Ubuntu 18.04+ / Debian 10+
    • CentOS 7+ / RHEL 7+
    • Fedora 30+
    • Arch Linux

最低硬件要求

  • 内存：至少 2GB RAM（推荐 4GB+）
  • 存储：100MB 可用磁盘空间
  • 网络：互联网连接（用于 API 调用和自动更新）

安装方法

方法一：官方安装脚本（推荐）

适用于所有平台，自动检测系统并下载对应版本

Windows (PowerShell)

irm https://claude.ai/install.ps1 | iex

macOS / Linux (Bash)

curl -fsSL https://claude.ai/install.sh | sh
提示：安装脚本会自动：
  • 检测你的操作系统和架构
  • 下载最新版本的原生二进制文件
  • 安装到用户目录（~/.local/bin）
  • 配置 PATH 环境变量

方法二：手动下载安装

适用于需要特定版本或离线安装的场景

1. 访问官方下载页面

打开浏览器访问：https://claude.ai/download

2. 选择对应平台下载

  • Windows：claude-windows-x64.exe
  • macOS：
    • Intel Mac：claude-macos-x64.tar.gz
    • Apple Silicon：claude-macos-arm64.tar.gz
    • Universal：claude-macos-universal.tar.gz
  • Linux：claude-linux-x64.tar.gz

3. Windows 安装步骤

# 下载完成后 ./claude-windows-x64.exe

按照安装向导完成安装，默认会添加到 PATH

4. macOS 安装步骤

# 解压文件 tar -xzf claude-macos-*.tar.gz # 移动到系统路径 sudo mv claude /usr/local/bin/ # 添加执行权限 chmod +x /usr/local/bin/claude

5. Linux 安装步骤

# 解压文件 tar -xzf claude-linux-x64.tar.gz # 移动到系统路径 sudo mv claude /usr/local/bin/ # 添加执行权限 chmod +x /usr/local/bin/claude

方法三：包管理器安装

macOS (Homebrew)

# 安装 brew install claude-code # 更新 brew upgrade claude-code

Windows (Scoop)

# 添加 bucket scoop bucket add claude # 安装 scoop install claude-code

Linux (APT)

# 添加 GPG 密钥 curl -fsSL https://claude.ai/gpg.key | sudo apt-key add - # 添加仓库 echo "deb https://claude.ai/apt stable main" | sudo tee /etc/apt/sources.list.d/claude.list # 安装 sudo apt update sudo apt install claude-code

配置说明

验证安装

API 配置

原生版本与 npm 版本使用相同的 API 配置方式

方式一：环境变量（推荐）

# Windows (PowerShell) $env:ANTHROPIC_API_KEY="sk-ant-api03-你的密钥" $env:ANTHROPIC_AUTH_TOKEN="sk-ant-api03-你的密钥" # macOS / Linux export ANTHROPIC_API_KEY="sk-ant-api03-你的密钥" export ANTHROPIC_AUTH_TOKEN="sk-ant-api03-你的密钥"

方式二：配置文件

创建配置文件 ~/.claude/config.json：

{ "apiKey": "sk-ant-api03-你的密钥", "authToken": "sk-ant-api03-你的密钥", "baseUrl": "https://api.anthropic.com", "model": "claude-3-sonnet-20241022" }

配置文件详解

配置项 说明 默认值
apiKey Anthropic API 密钥 必填
authToken 认证令牌 同 apiKey
baseUrl API 服务器地址 https://api.anthropic.com
model 默认模型 claude-3-sonnet-20241022
maxTokens 最大输出 token 数 4096
temperature 生成温度（0-1） 0.7

常见问题

Q1: Windows 显示安全警告

问题：运行 .exe 文件时 Windows Defender 提示安全风险

解决：

  • 点击"更多信息"
  • 选择"仍要运行"
  • 或将文件添加到 Windows Defender 白名单

Q2: Linux/macOS 权限被拒绝

问题：执行 claude 命令时提示 permission denied

解决：

chmod +x /usr/local/bin/claude

Q3: 命令找不到

问题：输入 claude 命令提示 command not found

解决：

  • 确认安装路径已添加到 PATH
  • Windows: 检查 %USERPROFILE%\.local\bin
  • macOS/Linux: 检查 ~/.local/bin
  • 重新打开终端

Q4: API 认证失败

问题：运行时提示认证错误

解决：

  • 检查 API Key 是否正确（以 sk-ant-api03- 开头）
  • 确认环境变量或配置文件已设置
  • 运行 claude doctor 检查配置

Q5: 自动更新失败

问题：claude update 命令执行失败

解决：

  • 手动下载最新版本替换旧文件
  • 检查网络连接和防火墙设置
  • 使用管理员权限运行

性能优化建议

  • 使用 SSD：虽然不是必须，但 SSD 可以进一步提升启动速度
  • 配置代理：如果网络环境受限，配置 HTTP 代理：
    export HTTP_PROXY=http://proxy.example.com:8080 export HTTPS_PROXY=http://proxy.example.com:8080
  • 缓存管理：Claude 会自动缓存常用数据，清理缓存：
    claude cache clear
