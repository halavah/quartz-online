# 常见问题

本页面收集了用户在使用 TechVerse 博客系统时遇到的常见问题及解决方案。

## 🚀 安装与设置

### Q: Node.js 版本要求是什么？
**A**: TechVerse 需要 Node.js 18.0 或更高版本。推荐使用 LTS 版本。

### Q: 安装依赖时出现权限错误怎么办？
**A**:
- **macOS/Linux**: 使用 `sudo npm install` 或配置 npm 全局目录权限
- **Windows**: 以管理员身份运行 PowerShell 或命令提示符

### Q: yarn 是否支持？
**A**: 支持，但推荐使用 npm。如果使用 yarn，确保删除 `package-lock.json`：
```bash
rm package-lock.json
yarn install
```

## 🏃‍♂️ 开发问题

### Q: `npm run dev` 启动失败
**A**: 检查以下几点：
1. 确认端口 3000 未被占用
2. 删除 `.next` 文件夹后重试
3. 清除 npm 缓存：`npm cache clean --force`

### Q: 页面刷新后样式丢失
**A**:
1. 检查 `tailwind.config.ts` 的 content 配置
2. 确认 `globals.css` 包含了 Tailwind 指令
3. 尝试强制刷新（Ctrl+F5 或 Cmd+Shift+R）

### Q: TypeScript 类型错误
**A**:
1. 运行 `npm run type-check` 查看具体错误
2. 确保 `tsconfig.json` 配置正确
3. 尝试删除 `.next/types` 文件夹

## 📝 内容管理

### Q: 如何添加新文章？
**A**:
1. 在 `public/` 目录下创建 HTML 文件
2. 在 `data/articles.json` 中添加文章信息
3. 重新构建：`npm run build`

### Q: 文章图片不显示
**A**:
1. 确认图片路径正确（相对于 HTML 文件）
2. 图片应放在 `public/` 目录下
3. `next.config.ts` 中设置 `images.unoptimized: true`

### Q: 文章中的 CSS 样式不生效
**A**:
- 检查样式是否内联在 HTML 中
- 确认选择器没有与全局样式冲突
- 使用更高的特异性或 `!important`

### Q: 如何使用代码高亮？
**A**: 在 HTML 文件中引入代码高亮库，例如 Prism.js：
```html
<link href="https://cdnjs.cloudflare.com/ajax/libs/prism/1.24.1/themes/prism-tomorrow.min.css" rel="stylesheet" />
<script src="https://cdnjs.cloudflare.com/ajax/libs/prism/1.24.1/prism.min.js"></script>
<script>Prism.highlightAll();</script>
```

## 🎨 样式与设计

### Q: 如何修改网站配色？
**A**: 编辑 `tailwind.config.ts` 中的 `colors` 配置：
```typescript
theme: {
  extend: {
    colors: {
      primary: {
        // 自定义主色调
      }
    }
  }
}
```

### Q: 如何更换字体？
**A**: 在 `app/globals.css` 中添加：
```css
@import url('https://fonts.googleapis.com/css2?family=YourFont&display=swap');
body {
  font-family: 'YourFont', sans-serif;
}
```

### Q: 移动端适配问题
**A**:
1. 确保添加了 viewport meta 标签
2. 使用 Tailwind 的响应式前缀（`sm:`, `md:`, `lg:`）
3. 测试不同屏幕尺寸的显示效果

## 🚀 部署问题

### Q: Vercel 部署失败
**A**:
1. 检查构建日志中的错误信息
2. 确认 `vercel.json` 配置正确
3. 确保所有依赖都在 `package.json` 中

### Q: GitHub Pages 部署后路由 404
**A**:
1. 在 `next.config.ts` 中添加 `basePath: '/repo-name'`
2. 添加 404 重定向规则
3. 使用 `exportTrailingSlash: true`

### Q: Nginx 部署后刷新页面 404
**A**: 配置 Nginx 重定向规则：
```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

### Q: 静态文件部署后样式丢失
**A**:
1. 确认 CSS 文件路径正确
2. 检查是否启用了 gzip 压缩
3. 确认 `assetPrefix` 配置（如果有）

## 🔧 构建问题

### Q: `npm run build` 失败
**A**:
1. 检查错误日志
2. 确保内存充足（至少 4GB）
3. 尝试删除 `.next` 和 `node_modules` 后重新安装

### Q: 构建时间过长
**A**:
1. 使用 `webpack-bundle-analyzer` 分析打包大小
2. 优化图片和大型依赖
3. 启用增量构建

### Q: 构建后的文件太大
**A**:
1. 使用代码分割
2. 压缩图片资源
3. 移除未使用的依赖

## 📊 性能问题

### Q: 首页加载慢
**A**:
1. 减少首页文章数量
2. 压缩图片
3. 使用 CDN 加速静态资源
4. 启用 Gzip 压缩

### Q: Lighthouse 评分低
**A**:
1. 优化图片（使用 WebP 格式）
2. 移除阻塞渲染的 JavaScript
3. 使用预加载关键资源
4. 添加适当的缓存策略

### Q: 移动端性能差
**A**:
1. 减少动画效果
2. 优化触控事件
3. 使用合适的图片尺寸
4. 启用浏览器缓存

## 🔍 搜索引擎优化

### Q: 如何添加 SEO 元信息？
**A**: 在每个 HTML 文章的 `<head>` 中添加：
```html
<meta name="description" content="文章描述">
<meta name="keywords" content="关键词1,关键词2">
<meta property="og:title" content="文章标题">
<meta property="og:description" content="文章描述">
```

### Q: 如何生成 sitemap？
**A**: 创建脚本生成 `public/sitemap.xml`：
```javascript
// scripts/generate-sitemap.js
const fs = require('fs');
const articles = require('../data/articles.json');

const sitemap = `<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://yourdomain.com</loc>
  </url>
  ${articles.articles.map(article => `
  <url>
    <loc>https://yourdomain.com/article/${article.htmlFile.replace('.html', '')}</loc>
  </url>`).join('')}
</urlset>`;

fs.writeFileSync('public/sitemap.xml', sitemap);
```

## 🔄 迁移问题

### Q: 从其他博客系统迁移
**A**:
1. 导出文章内容为 HTML 格式
2. 按要求组织文件结构

### Q: 从 WordPress 迁移

2. 提取元数据创建配置

## 🐛 常见错误代码

### Error: ENOENT: no such file or directory
**原因**: 文件路径错误或文件不存在
**解决**:
1. 检查 `htmlFile` 路径是否正确
2. 确认文件确实存在于 `public/` 目录下

### Error: Module not found
**原因**: 依赖未安装
**解决**:
```bash
npm install
# 或
rm -rf node_modules package-lock.json
npm install
```

### TypeError: Cannot read property 'map' of undefined
**原因**: `articles.json` 格式错误
**解决**: 检查 JSON 格式，确保 `articles` 是数组

### Build optimization failed
**原因**: 内存不足或代码错误
**解决**:
1. 增加内存限制：`NODE_OPTIONS="--max-old-space-size=4096" npm run build`
2. 检查代码语法错误

## 🛠️ 调试技巧

### 1. 开启调试模式
```bash
NODE_OPTIONS='--inspect' npm run dev
```

### 2. 查看详细日志
```bash
DEBUG=* npm run build
```

### 3. 分析打包
```bash
npm install --save-dev webpack-bundle-analyzer
npx webpack-bundle-analyzer .next/static/chunks/*.js
```

### 4. 清除缓存
```bash
# Next.js 缓存
rm -rf .next

# npm 缓存
npm cache clean --force

# 构建缓存
rm -rf node_modules/.cache
```
