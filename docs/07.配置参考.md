# 配置参考

本文档详细介绍了 TechVerse 博客系统的各项配置选项。

## 📁 核心配置文件

### 1. articles.json - 文章配置

位置：`/data/articles.json`

```json
{
  "config": {
    "siteName": "TechVerse",
    "siteDescription": "探索技术的无限可能"
  },
  "articles": [
    {
      "title": "文章标题",
      "description": "文章描述",
      "htmlFile": "tech/article.html"
    }
  ]
}
```

#### 配置说明

##### config 部分

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `siteName` | string | ✅ | 网站名称，显示在页面标题和导航栏 |
| `siteDescription` | string | ✅ | 网站描述，显示在导航栏下方 |

##### articles 部分

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | ✅ | 文章标题 |
| `description` | string | ✅ | 文章描述，显示在列表页 |
| `htmlFile` | string | ✅ | HTML 文件路径（相对于 public） |

## ⚡ 性能配置

### 构建优化

```javascript
// next.config.ts
const nextConfig = {
  // 启用压缩
  compress: true,

  // 启用实验性功能
  experimental: {
    optimizeCss: true,
  },

  // 构建时优化
  webpack: (config, { isServer }) => {
    if (!isServer) {
      config.optimization.splitChunks = {
        chunks: 'all',
        cacheGroups: {
          vendor: {
            test: /[\\/]node_modules[\\/]/,
            name: 'vendors',
            chunks: 'all',
          },
        },
      };
    }
    return config;
  },
};
```

## 🔒 安全配置

### CSP 配置

```javascript
// next.config.ts
const nextConfig = {
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          {
            key: 'Content-Security-Policy',
            value: [
              "default-src 'self'",
              "script-src 'self' 'unsafe-eval' 'unsafe-inline' https://www.googletagmanager.com",
              "style-src 'self' 'unsafe-inline'",
              "img-src 'self' data: https:",
              "font-src 'self'",
              "connect-src 'self'",
            ].join('; '),
          },
          {
            key: 'X-Frame-Options',
            value: 'DENY',
          },
          {
            key: 'X-Content-Type-Options',
            value: 'nosniff',
          },
          {
            key: 'Referrer-Policy',
            value: 'strict-origin-when-cross-origin',
          },
        ],
      },
    ];
  },
};
```

   ```bash
   npm run dev
   ```
3. 重新构建生产版本：

## 🔍 配置优化建议

1. **按需加载**：只在需要时引入大型库
2. **代码分割**：合理配置 webpack 分包
3. **缓存策略**：配置适当的缓存头
4. **压缩优化**：启用 gzip/brotli 压缩
5. **图片优化**：使用适当的格式和尺寸
