## 1. 基础命令 - 5分钟上手

先从最简单的开始,这些命令能满足你80%的需求。

| 命令 | 功能说明 |
|------|----------|
| `claude` | 启动Claude Code交互模式,开始与AI对话进行编程 |
| `/model` | 选择不同的AI模型 (Sonnet, Opus, Haiku),根据任务复杂度选择 |
| `/resume` | 恢复上次对话,继续之前未完成的工作 |
| `/mcp` | 查看和管理MCP (Model Context Protocol) 服务器配置 |
| `/compact` | 压缩当前对话历史,减少token使用,优化性能 |
| `/init` | 初始化项目配置,设置Claude Code在当前项目中的工作环境 |
| `/clear` | 清理当前会话,开始全新的对话 |
| `/cost` | 查看当前会话的token使用量和费用统计 |

> 💡 **小贴士**: 使用 `/model` 切换到 Haiku 模型可以大幅降低简单任务的成本,而 Opus 模型适合复杂的架构设计任务。

## 2. 实用技巧

这些技巧能让你的工作效率翻倍。

- **Control + C** - 终止当前输出
  当Claude的回答太长或方向错误时,立即按下此快捷键终止输出,节省时间和token

- **Alt/Command + V** - 粘贴图片到终端
  直接在对话中粘贴截图、设计稿或报错界面,Claude能识别图片内容并提供针对性建议

- **CLAUDE.md** - 项目全局说明文档
  在项目根目录创建 `CLAUDE.md` 文件,写入项目架构、编码规范、常用命令等信息。Claude会自动读取这个文件,就像给AI配了一份"项目使用手册"

> 📂 **CLAUDE.md 示例内容**:
> 项目技术栈: React + TypeScript + Vite
> 代码风格: ESLint + Prettier
> 测试命令: npm test
> 部署命令: npm run build

## 3. 提问技巧

好的提问方式能让Claude更准确地理解你的需求。

- **明确具体**
  ❌ 不好: "有问题"
  ✅ 好的: "登录接口返回500错误,请求体为 {username: 'test', password: '123456'},后端日志显示数据库连接超时"

- **提供上下文**
  ❌ 不好: "帮我写个组件"
  ✅ 好的: "我在用React + TypeScript开发一个电商网站,需要一个商品卡片组件,要展示商品图片、标题、价格和购买按钮"

- **分步骤**
  ❌ 不好: "帮我做一个完整的用户系统"
  ✅ 好的:
  1. 先帮我设计数据库表结构
  2. 然后实现用户注册接口
  3. 接着实现登录和JWT token验证
  4. 最后添加用户信息修改功能

- **包含报错信息**
  出现错误时,不要只说"报错了",要把完整的错误堆栈、报错行数、相关代码一起提供给Claude

- **说明期望结果**
  除了描述问题,还要说明你期望的结果是什么,这样Claude能更好地调整解决方案

> ⚠️ **注意**: 避免一次性提出太多需求。如果任务复杂,建议分解成多个小任务逐步完成,这样能获得更准确的结果。
